# 공통 상세 업무기능분해도

## 1. 문서 개요

### 1.1 목적

본 문서는 BMS(Business Management System)의 사용자 인증, 대시보드와 여러 업무영역에서 공통으로 사용하는 업무대상, 첨부파일, 메모 및 알림 기능을 정의한다.

공통 기능은 시스템 진입과 공통 현황 조회를 지원하고 계약, 프로젝트, 영업과 인력 등 개별 업무 화면에서 재사용하며 독립된 업무 데이터를 중복 생성하지 않는다.

### 1.2 적용 범위

- 사용자 로그인·로그아웃 및 인증 세션 관리
- 로그인 사용자의 권한 범위에 따른 일반 사용자 대시보드 조회
- 업무대상ID 기반 첨부파일 등록, 조회, 다운로드, 삭제 및 연결 검증
- 업무대상별 메모 조회, 등록, 수정 및 삭제
- 사용자별 알림 조회 및 읽음 처리
- 첨부파일 저장 및 접근에 필요한 기본 검증

# 4. 주요 업무 규칙

| 구분 | 업무 규칙 |
| --- | --- |
| 업무대상 식별 | 첨부파일과 메모는 업무대상ID로 실제 업무대상을 필수 참조한다. 업무대상 유형은 입력받지 않고 업무대상에서 조회한다. |
| 업무대상 범위 | 계약, 프로젝트, 영업기회, 계약문서, 사업공고, 제안과 인력에 공통 기능을 제공하며 다른 엔터티는 별도 승인 없이 업무대상으로 등록하지 않는다. |
| 업무대상 상태 | 논리삭제된 업무대상에는 신규 첨부파일과 메모를 등록할 수 없으며 기존 데이터 조회는 보존·권한 정책에 따른다. |
| 업무대상 유형 | 업무대상유형코드와 실제 업무 엔터티가 일치해야 하며 유형에 대응하는 접근권한 판정기가 없거나 대상 엔터티가 없으면 처리를 차단한다. |
| 파일 소속 | 하나의 첨부파일은 하나의 업무대상에만 소속하고 등록 후 업무대상ID를 변경하지 않으며 이미 등록된 파일을 다른 업무대상에 다시 연결하지 않는다. |
| 파일 크기 | 시스템에서 설정한 최대 파일 크기를 초과하는 파일은 등록할 수 없다. |
| 파일 확장자 | 시스템에서 허용한 확장자만 등록할 수 있으며 파일명 확장자와 실제 파일 형식을 함께 검증한다. 실행 파일 등 금지 형식은 등록할 수 없다. |
| 저장경로 | 저장경로와 저장파일명은 시스템이 생성하며 사용자가 직접 지정할 수 없다. 경로 정규화 후 허용된 저장영역 밖의 경로는 사용할 수 없다. |
| 접근권한 | 첨부파일 조회·다운로드·등록·삭제 권한은 연결된 업무대상의 조회·관리 권한을 따른다. |
| 파일 삭제 | 계약문서 등 다른 업무정보가 파일을 참조하고 있으면 삭제를 차단하고 참조 해제 후 삭제하도록 안내한다. |
| 계약문서 PDF | 계약문서는 계약과 별개의 업무대상으로 등록하고 계약문서가 참조하는 PDF 첨부파일의 업무대상ID는 계약문서의 업무대상ID와 일치해야 한다. |
| 메모 권한 | 메모 조회·등록 권한은 대상 업무 권한을 따르며 수정·삭제는 작성자 또는 관리권한이 있는 사용자만 가능하다. |
| 알림 권한 | 알림은 수신자 본인만 조회하고 읽음 처리할 수 있다. |
| 대시보드 권한 | 대시보드는 로그인 사용자가 접근할 수 있으며 사용자의 역할과 업무 권한 범위에 해당하는 정보만 제공한다. |
| 로그인 허용 | 활성 상태의 사용자만 로그인할 수 있다. 로그인은 5회까지 연속 실패를 허용하고 5회를 초과하는 6번째 실패에서 계정을 비활성화한다. |
| 인증 세션 | 로그아웃하거나 설정된 유휴시간을 초과한 세션은 무효화하며 이후 요청은 다시 인증해야 한다. |
| 세션 유휴시간 | 로그인 후 15분 동안 사용자 작업이 없으면 자동 로그아웃한다. |

# 5. 예외 및 오류처리 기준

| 상황 | 처리 기준 |
| --- | --- |
| 최대 파일 크기 초과 | 등록을 차단하고 허용 가능한 최대 크기를 안내한다. |
| 허용되지 않은 확장자 또는 파일 형식 | 등록을 차단하고 허용 가능한 파일 형식을 안내한다. |
| 잘못된 저장경로 | 파일 저장을 중단하고 보안 오류로 기록한다. |
| 업무대상 없음 | 첨부파일 연결 또는 메모 등록을 차단한다. |
| 삭제된 업무대상 | 신규 첨부파일·메모 등록을 차단하고 기존 데이터는 조회·보존 정책에 따라 처리한다. |
| 업무대상 유형 불일치 | 업무대상 유형과 실제 엔터티 또는 권한 판정기가 일치하지 않으면 처리를 차단하고 정합성 오류로 기록한다. |
| 업무대상 접근권한 없음 | 파일 조회·다운로드·등록·삭제 및 메모 처리를 차단한다. |
| 첨부파일 중복 소속 | 이미 다른 업무대상에 소속된 파일의 재연결을 차단한다. |
| 계약문서와 PDF 업무대상 불일치 | 계약문서 등록·교체를 차단하고 동일한 계약문서 업무대상으로 파일을 다시 등록하도록 안내한다. |
| 참조 중인 파일 삭제 | 파일을 참조하는 업무정보가 있음을 안내하고 삭제를 차단한다. |
| 다른 사용자의 알림 처리 | 알림 조회 및 읽음 처리를 차단한다. |
| 로그인 정보 불일치 | 사용자ID 또는 비밀번호 중 어느 항목이 잘못되었는지 구분하지 않는 공통 오류 메시지를 제공하고 실패 이력을 기록한다. |
| 비활성 계정 | 로그인을 차단하고 관리자에게 활성화를 요청하도록 안내한다. |
| 인증 세션 만료 | 요청을 처리하지 않고 재로그인이 필요함을 안내한다. |

# 6. 데이터 및 관계

| 구분 | 내용 |
| --- | --- |
| 주요 엔터티 | 업무대상, 알림, 첨부파일, 메모 |
| 사용자 1 : N 알림 | 알림의 수신자ID로 알림 수신 사용자를 식별한다. |
| 업무대상 1 : N 첨부파일 | 첨부파일의 업무대상ID가 업무대상을 실제 FK로 참조하며 하나의 파일은 하나의 업무대상에만 소속한다. |
| 업무대상 1 : N 메모 | 메모의 업무대상ID가 업무대상을 실제 FK로 참조한다. |
| 업무대상 1 : 0..1 적용 엔터티 | 계약, 프로젝트, 영업기회, 계약문서, 사업공고, 제안과 인력은 업무대상ID를 필수·고유 참조한다. |
| 첨부파일 1 : 0..1 계약문서 | 모든 계약문서는 PDF 파일 하나를 필수·고유 참조하고 두 엔터티의 업무대상ID가 일치해야 한다. |
| 사용자 1 : N 접속로그 | 로그인 성공·실패 및 로그아웃 이력은 시스템관리의 사용자와 접속로그를 사용한다. |
