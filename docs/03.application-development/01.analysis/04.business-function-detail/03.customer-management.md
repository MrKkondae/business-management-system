# 고객관리 상세 업무기능분해도

## 1. 문서 개요

### 1.1 목적

본 문서는 BMS(Business Management System)의 고객관리 업무에 대한 상세 업무기능을 정의한다.

고객관리 기능은 고객사 및 고객 담당자의 기본정보와 고객 대상 영업활동을 관리하여 영업관리, 계약관리 및 프로젝트 기준정보를 참조하는 후속 업무에 기준 정보를 제공한다.

# 3. 상세 업무기능

## 3.1 고객사관리

### 3.1.1 고객사 기본관리

| 기능ID            | 기능명    | 설명                   |
| --------------- | ------ | -------------------- |
| BFD-03-01-01-01 | 고객사 조회 | 고객사 목록 및 상세정보를 조회한다. |
| BFD-03-01-01-02 | 고객사 등록 | 고객명과 고객유형을 필수 입력하여 신규 고객사를 등록하며 고객상태는 `ACTIVE`를 기본값으로 설정한다. 사업자등록번호가 있으면 삭제된 고객사를 포함하여 중복을 허용하지 않는다. |
| BFD-03-01-01-03 | 고객사 수정 | 고객사 정보를 수정한다.        |
| BFD-03-01-01-04 | 고객사 삭제 | 고객담당자, 영업활동 및 후속 업무의 참조 여부를 확인한 후 고객사를 논리삭제한다. 참조 이력은 보존한다. |

### 3.1.2 고객사 상태관리

| 기능ID            | 기능명      | 설명           |
| --------------- | -------- | ------------ |
| BFD-03-01-02-01 | 고객상태 변경  | 고객상태를 변경한다.  |
| BFD-03-01-02-02 | 고객사 활성화  | 삭제되지 않은 고객사의 고객상태코드를 `ACTIVE`로 변경하여 신규 업무에서 선택할 수 있도록 한다. |
| BFD-03-01-02-03 | 고객사 비활성화 | 고객상태코드를 `INACTIVE`로 변경하여 신규 업무의 선택 대상에서 제외하되 기존 업무 이력은 유지한다. |

## 3.3 영업활동관리

영업활동은 고객과의 모든 활동을 통합 관리한다.

활동 유형은 **공통코드(ACTIVITY_TYPE)** 로 관리하며 방문, 전화, 메일, 미팅, 제안, 기타 등을 선택할 수 있다.

### 3.3.1 영업활동관리

| 기능ID            | 기능명     | 설명             |
| --------------- | ------- | -------------- |
| BFD-03-03-01-01 | 영업활동 조회 | 고객 영업활동을 조회한다. |
| BFD-03-03-01-02 | 영업활동 등록 | 고객사, 활동유형, 활동일시와 활동담당사용자를 필수 입력하여 영업활동을 등록한다. 고객담당자를 선택한 경우 해당 활동 고객사와 동일한 고객사 소속인지 검증한다. |
| BFD-03-03-01-03 | 영업활동 수정 | 영업활동을 수정한다.    |
| BFD-03-03-01-04 | 영업활동 삭제 | 영업활동을 삭제한다.    |

#### 활동유형(공통코드)

| 코드       | 설명 |
| -------- | -- |
| VISIT    | 방문 |
| CALL     | 전화 |
| MEETING  | 미팅 |
| EMAIL    | 메일 |
| PROPOSAL | 제안 |
| ETC      | 기타 |

# 4. 주요 업무 규칙

| 구분 | 업무 규칙 |
| --- | --- |
| 고객 기본상태 | 신규 고객사의 고객상태코드는 `ACTIVE`를 기본값으로 사용한다. |
| 담당자 기본상태 | 신규 고객담당자의 고객담당자상태코드는 `ACTIVE`를 기본값으로 사용한다. |
| 상태와 삭제 구분 | `ACTIVE`와 `INACTIVE`는 업무 선택 가능 상태를 나타내고 삭제여부는 논리삭제를 나타낸다. 비활성화 처리에 삭제여부를 사용하지 않는다. |
| 비활성 정보 참조 | 비활성 고객사와 고객담당자는 신규 업무의 선택 대상에서 제외하지만 기존 영업활동과 후속 업무 이력 조회에서는 명칭을 확인할 수 있어야 한다. |
| 사업자등록번호 고유성 | 사업자등록번호는 값이 있는 경우 삭제 여부와 관계없이 중복 등록하거나 재사용할 수 없다. |
| 고객담당자 소속 | 영업활동에 고객담당자를 지정하면 고객담당자가 해당 영업활동의 고객사에 소속되어 있어야 한다. |
| 영업활동 필수값 | 영업활동 등록 시 고객사, 활동유형, 활동일시와 활동담당사용자를 필수로 입력한다. |
| 일시 저장 | 활동일시는 UTC로 저장하고 화면에서 `Asia/Seoul` 기준으로 표시한다. |
| 논리삭제 | 고객사, 고객담당자와 영업활동의 삭제는 `DEL_YN='Y'`로 처리하며 참조 중인 데이터와 업무 이력을 물리삭제하지 않는다. |

# 5. 업무 연계

고객관리 기능은 다음 업무와 연계된다.

| 업무     | 연계 내용                 |
| ------ | --------------------- |
| 영업관리   | 영업기회 및 제안 대상 고객 정보 제공 |
| 계약관리   | 계약 대상 고객 정보 제공        |
| 프로젝트 기준정보 참조 업무 | 프로젝트 발주기관 정보 제공       |
| 유지관리   | 유지보수 대상 고객 정보 제공      |
| 공통     | 고객 검색 및 고객 정보 조회      |
