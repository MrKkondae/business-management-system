# 영업관리 상세 업무기능분해도

## 1. 문서 개요

### 1.1 목적

본 문서는 BMS의 영업관리 업무에 대한 상세 업무기능을 정의한다.

영업관리 기능은 사용자가 직접 등록한 사업공고와 조달청 나라장터 OpenAPI를 통해 자동 수집한 사업공고를 함께 관리하고, 담당자 검토 후 영업기회, 사업성 분석, 제안 및 수주 단계로 전환하는 기능을 제공한다.

# 4. 사업공고 자동수집 처리 흐름

```text
외부연계 설정 및 서비스키 참조 확인
        ↓
사용 중인 수집조건 조회
        ↓
마지막 성공 수집시점 기준 조회기간 계산
        ↓
업무구분별 나라장터 API 페이지 호출
        ↓
응답 원문·API 호출로그 저장
        ↓
연계 명세의 모든 응답 항목을 연계수신에 저장
        ↓
필수값 검증 및 발주기관·키워드 필터 적용
        ↓
공고번호 + 차수 + 변경일시/원문해시 중복·변경 판정
        ↓
사업공고 신규 등록 / 변경·재공고·취소 반영 / 중복 제외
        ↓
연계 처리상태·반영일시와 수집이력 결과 갱신
        ↓
담당자 관심·검토·제외 처리
        ↓
담당자 선택에 따른 영업기회 전환
```

- 증분 조회 시작시점에는 외부시스템의 지연 반영에 대비한 중첩 구간을 적용한다.
- 전체 페이지가 성공한 경우에만 수집조건의 마지막 성공 수집일시를 갱신한다.
- 수집 원문은 버전별로 보존하고 사용자 보완정보를 자동수집 값으로 덮어쓰지 않는다.
- 원문과 연계수신은 사업공고 반영 전에 먼저 커밋하고 사업공고 반영과 연계 완료상태 갱신은 하나의 트랜잭션으로 처리한다.
- 조달청 응답 필드는 표준단어·표준용어 적용 예외로 하고 연계 명세의 영문명과 국문명을 원형 그대로 저장한다.
- 영업기회 전환은 단일 트랜잭션으로 처리하고 공고별 연계정보의 유일성을 보장한다.

# 5. 공통코드 후보

| 코드그룹 후보 | 코드값 후보 | 용도 |
| --- | --- | --- |
| 공고업무구분코드 | THNG, SERVC, CNSTWK, FRGCPT, ETC | 물품, 용역, 공사, 외자, 기타 구분 |
| 외부공고상태코드 | NEW, CHANGED, READVERTISED, CANCELLED | 신규, 변경, 재공고, 취소 상태 |
| 수집실행유형코드 | SCHEDULED, MANUAL, RETRY | 정기, 즉시, 재처리 실행 구분 |
| 수집처리상태코드 | RUNNING, SUCCESS, PARTIAL_FAILED, FAILED | 수집 실행 결과 |
| 연계처리구분코드 | INSERT, UPDATE, CANCEL, DUPLICATE | 개별 연계수신의 사업공고 반영 구분 |
| 연계처리상태코드 | RECEIVED, VALIDATED, REFLECTED, DUPLICATE, FAILED, RETRYING | 개별 연계수신의 검증·반영·재처리 상태 |
| 재처리상태코드 | NONE, PENDING, PROCESSING, SUCCESS, EXHAUSTED | 오류 재처리 상태 |
| 키워드구분코드 | INCLUDE, EXCLUDE | 포함 및 제외 키워드 구분 |
| API호출결과코드 | SUCCESS, BUSINESS_ERROR, AUTH_ERROR, NETWORK_ERROR, LIMIT_EXCEEDED | API 호출 결과 구분 |

나라장터 원천 코드와 BMS 공통코드는 직접 혼용하지 않고 매핑하여 사용한다.

# 6. 예외 및 오류처리 기준

| 상황 | 처리 기준 |
| --- | --- |
| 인증정보 누락·오류 | 수집을 즉시 중단하고 인증값을 제외한 오류만 기록하며 마지막 성공 수집일시는 갱신하지 않는다. |
| 연결 실패·타임아웃·HTTP 5xx | 설정된 횟수만큼 재시도하고 초과 시 실패 또는 부분실패로 기록한다. |
| 호출량 제한 초과 | 현재 실행을 중단하고 재처리 대상으로 등록한다. |
| 일부 페이지 처리 실패 | 성공 항목은 멱등 저장하고 실행은 부분실패로 기록하며 마지막 성공 수집일시는 갱신하지 않는다. |
| 필수 식별값 누락 | 원문과 연계수신은 보존하고 연계처리상태를 FAILED로 기록하며 나머지 항목은 계속 처리한다. |
| 동일 공고번호·차수 재수신 | 원문 해시와 외부 수정일시가 같으면 중복 처리하고 다르면 변경 처리한다. |
| 변경·재공고·취소 수신 | 새 원문과 연계수신을 추가한 후 사업공고 업무정보만 반영한다. |
| 사업공고 반영 실패 | 연계수신을 FAILED로 유지하고 저장된 수신값으로 재처리하며 OpenAPI를 불필요하게 재호출하지 않는다. |
| 전환 완료 공고 취소 | 생성된 영업기회는 삭제하지 않고 변경상태를 표시하여 담당자가 후속 조치하도록 한다. |
| 정기수집과 즉시수집 동시 실행 | 동일 수집조건에 실행 잠금을 적용하고 중복키로 이중 등록을 방지한다. |
| 영업기회 동시 전환 | 공고·영업기회 연계의 유일성 검증과 트랜잭션으로 중복 전환을 차단한다. |
| 로그 기록 | 서비스키, 인증 헤더 및 민감한 쿼리 값은 제거하거나 마스킹한다. |
