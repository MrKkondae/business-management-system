# 프로젝트관리 상세 업무기능분해도

## 1. 문서 개요

### 1.1 목적

본 문서는 v1.0 인력투입에 필요한 최소 프로젝트 기준정보의 상세 업무기능을 정의한다.

### 1.2 적용 범위

- 프로젝트 기본정보 수동 등록·조회·수정 및 논리삭제
- 프로젝트 기간과 계획·진행·연장·완료·취소 상태 관리
- 인력투입에서 사용하는 유효 프로젝트 조회 기준 제공
- 향후 계약관리 도입 시 선택적인 계약 연결 허용

일정, 이슈, 위험, 산출물, 진척률, 원가·손익 및 독립 프로젝트 운영관리는 v1.0 범위에서 제외한다.

## 3. 상세 업무기능

### 3.1 프로젝트 기준정보관리

#### 3.1.1 프로젝트 기본관리

| 기능 ID | 기능명 | 기능 설명 |
| --- | --- | --- |
| BFD-06-01-01-01 | 프로젝트 목록조회 | 프로젝트번호, 고객사, 프로젝트명, 상태, 기간과 PM을 조건으로 삭제되지 않은 프로젝트를 조회한다. |
| BFD-06-01-01-02 | 프로젝트 상세조회 | 프로젝트 기본정보, 최초·현재 종료일, 상태와 인력투입 참조 여부를 조회한다. |
| BFD-06-01-01-03 | 프로젝트 등록 | 프로젝트 업무대상과 프로젝트번호, 고객사, 명칭, 기간 및 상태를 하나의 처리 단위로 수동 등록한다. 최초종료일자와 현재 종료일자는 같은 값으로 시작한다. |
| BFD-06-01-01-04 | 프로젝트 수정 | 프로젝트명, 고객사, 유형과 PM 등 기본정보를 수정한다. 프로젝트번호와 최초종료일자는 변경하지 않는다. |
| BFD-06-01-01-05 | 프로젝트 삭제 | 인력투입 등 업무 참조가 없는 잘못 등록된 프로젝트를 논리삭제한다. |

#### 3.1.2 프로젝트 기간·상태관리

| 기능 ID | 기능명 | 기능 설명 |
| --- | --- | --- |
| BFD-06-01-02-01 | 프로젝트 상태변경 | 허용된 상태전이에 따라 프로젝트 상태를 계획, 진행, 완료 또는 취소로 변경한다. |
| BFD-06-01-02-02 | 프로젝트 기간연장 | 최초종료일자는 보존하고 현재 종료일자를 기존 값보다 이후로 변경한 뒤 상태를 연장으로 변경한다. |

## 4. 주요 업무규칙

- 프로젝트번호는 논리삭제 여부와 관계없이 중복하거나 재사용하지 않는다.
- 프로젝트 시작일자는 최초종료일자보다 늦을 수 없고 최초종료일자는 현재 종료일자보다 늦을 수 없다.
- 연장 상태에서는 현재 종료일자가 최초종료일자보다 이후여야 한다.
- 완료 또는 취소된 프로젝트는 신규 인력투입 대상으로 선택할 수 없다.
- 인력투입에서 참조 중인 프로젝트는 물리삭제하지 않는다.
- v1.0에서는 연장 횟수와 변경 전후 일자를 별도 이력으로 저장하지 않는다. 필요해지면 프로젝트 연장이력을 하위 엔터티로 추가한다.
