# 인력관리 상세 업무기능분해도(초안)

## 1. 문서 개요

### 1.1 목적

본 문서는 BMS(Business Management System)의 인력관리 업무에 대한 상세 업무기능을 정의한다.

인력관리 기능은 내부 직원과 외주인력의 공통 인력정보를 통합 관리하고, 직원 재직정보, 협력업체와 외주인력, 경력·학력·교육·상훈·기술자격·기술경력, 인력단가, 프로젝트 투입계획 및 월별 실적을 관리한다.

### 1.2 적용 원칙

- `인력`은 직원과 외주인력의 공통 상위 엔터티이며 모든 공통 업무 참조는 `인력ID`를 사용한다.
- 인력구분코드에 따라 `직원` 또는 `외주인력` 상세정보 중 하나만 유지한다.
- 직원과 외주인력은 경력·학력·교육·상훈·기술자격·기술경력·단가를 공통으로 관리한다.
- 첨부파일과 메모는 인력 업무대상 단위로만 관리하며 경력·학력·교육·상훈·기술자격별로 별도 연결하지 않는다.
- 사람 사용자 계정은 내부 직원만 선택적으로 연결할 수 있고 외주인력에는 사용자 계정을 생성하지 않는다.
- 프로젝트 기준정보는 계약관리에서 생성된 정보를 참조하며 독립 프로젝트관리 기능은 본 업무 범위에 포함하지 않는다.

# 4. 주요 업무 규칙

| 구분 | 업무 규칙 |
| --- | --- |
| 인력 하위유형 | 하나의 인력에는 직원 또는 외주인력 상세 중 하나만 존재해야 한다. |
| 업무대상 | 인력 등록 시 인력 업무대상과 인력 엔터티를 동일 트랜잭션에서 생성하며 첨부파일·메모는 인력의 업무대상ID를 사용한다. |
| 하위 데이터 첨부 | 인력근무경력, 인력학력, 인력교육, 인력상훈과 인력기술자격은 데이터로만 관리하고 독립 업무대상·첨부파일·메모를 생성하지 않는다. |
| 직원 식별 | 사원번호는 내부 직원 간 중복되지 않아야 한다. |
| 사용자 계정 | `사용자.직원ID`가 `직원.인력ID`를 선택적으로 참조하며 직원 한 명에는 최대 하나의 사용자 계정만 연결할 수 있다. |
| 외주인력 계정 | 외주인력은 BMS 사용자 계정의 연결 대상이 될 수 없다. |
| 협력업체 식별 | 사업자등록번호가 있는 협력업체는 사업자등록번호의 중복을 허용하지 않는다. |
| 인력단가 | 기준연월은 필수이며 같은 인력ID와 기준연월 조합은 한 번만 등록할 수 있다. |
| 프로젝트 참조 | 인력투입은 계약관리에서 생성한 최소 프로젝트 기준정보의 프로젝트ID를 필수 참조한다. |
| 투입기간 | 투입시작일자는 필수이며 투입종료일자보다 늦을 수 없고 프로젝트 기준기간을 벗어나는 경우 경고 또는 제한한다. |
| 중복투입 | 같은 프로젝트·인력의 투입기간 중복은 금지한다. 서로 다른 프로젝트의 동시 투입은 허용한다. 물리설계에서는 기간 중복 방지 제약 또는 동등한 트랜잭션 검증을 적용한다. |
| 월별 투입률 | 인력별 월간 계획 및 실적 투입률 합계는 시스템에서 설정한 상한을 초과할 수 없다. |
| 월별 실적 | 기준연월은 필수이며 같은 인력투입ID와 기준연월 조합은 한 번만 등록하고 기준연월은 인력투입 기간 안에 있어야 한다. |
| 실적확정 | 확정된 월별 실적은 수정하지 못하며 확정취소 권한과 절차를 거쳐야 한다. |
| 삭제정책 | 프로젝트 투입, 단가, 사용자 계정 등 업무 참조가 있는 인력은 물리 삭제하지 않고 논리삭제한다. 상태코드가 있는 엔터티는 상태 변경을 사용한다. |
| 개인정보 | 생년월일, 휴대전화번호, 이메일주소 등 개인정보는 권한이 있는 사용자만 조회·수정할 수 있다. |

## 7.1 주요 엔터티

| 구분 | 엔터티 |
| --- | --- |
| 인력 기본 | 업무대상, 인력, 직원, 외주인력, 협력업체 |
| 이력·역량 | 인력근무경력, 인력학력, 인력교육, 인력상훈, 인력기술자격, 인력기술경력 |
| 단가·투입 | 인력단가, 인력투입, 인력월별투입실적 |

## 7.2 주요 관계

| 관계 | 설명 |
| --- | --- |
| 인력 1 : 0..1 직원 | 내부 직원이면 인력ID를 공유하는 직원 상세정보를 가진다. |
| 인력 1 : 0..1 외주인력 | 외주인력이면 인력ID를 공유하는 외주인력 상세정보를 가진다. |
| 업무대상 1 : 0..1 인력 | 인력은 업무대상ID를 필수·고유 참조하며 첨부파일·메모는 해당 업무대상에 연결한다. |
| 협력업체 1 : N 외주인력 | 하나의 협력업체에 여러 외주인력이 소속될 수 있다. |
| 인력 1 : N 인력이력 | 인력은 경력·학력·교육·상훈·기술자격·기술경력을 여러 건 가질 수 있다. |
| 인력 1 : N 인력단가 | 인력은 기준연월별 단가 이력을 가진다. |
| 프로젝트 1 : N 인력투입 | 프로젝트에는 여러 직원 또는 외주인력이 투입될 수 있다. |
| 인력투입 1 : N 인력월별투입실적 | 투입 건별로 월간 계획 및 실적을 관리한다. |

# 9. 후속 검토 사항

- 직원의 조직·직위·재직상태 변경이력을 별도 엔터티로 관리할지 검토한다.
- 외주인력의 소속 협력업체 변경이력을 별도로 관리할지 검토한다.
- 외주계약 기간·계약형태·계약상태를 외주인력에 둘지 별도 외주계약 엔터티로 분리할지 검토한다.
- 월별 총투입률 상한과 초과 승인 허용 여부를 확정한다.
- 실적 확정 및 확정취소 권한과 마감 절차를 확정한다.
- 개인정보 보존기간, 마스킹 및 파기 기준을 확정한다.
