# 계약관리 상세 업무기능분해도(초안)

## 1. 문서 개요

### 1.1 목적

본 문서는 BMS(Business Management System)의 계약관리 업무에 대한 상세 업무기능을 정의한다.

계약관리 기능은 하나의 계약을 기준으로 계약 기본정보, 계약서에 기재되는 기관, 계약 참여업체와 지분, 실제 계약문서 PDF 및 계약 변경이력을 하나의 계약관리 화면에서 통합 관리하고, 인력투입·원가·매출 업무에서 참조할 프로젝트 기준정보를 생성한다.

### 1.2 적용 원칙

- 계약 기본정보는 고객사와 선택적으로 수주정보를 참조한다.
- 영업기회·제안·수주 기능이 후속 구축 대상이므로 1차 구축에서는 수주 없이 계약을 직접 등록할 수 있다.
- 계약정보, 계약기관, 계약업체, 계약문서 PDF 및 계약변경이력은 하나의 계약 집합으로 관리한다.
- 계약문서는 계약과 별개의 업무대상으로 등록하고 계약 내용을 다시 구조화하지 않으며 실제 계약서·변경계약서 PDF의 문서정보와 동일 업무대상의 첨부파일 참조를 저장한다.
- 계약금액, 계약기간, 납품기한 등 주요 계약조건을 변경할 때는 계약변경이력을 남긴다.
- 독립 프로젝트관리 화면은 구현하지 않고 계약관리 화면에서 프로젝트 기준정보를 생성한다.

## 2. 업무기능 구성

| L2 | L3 | 화면 구성 |
| --- | --- | --- |
| 계약관리 | 계약정보관리 | 계약 기본정보 영역 |
| 계약관리 | 계약기관관리 | 계약기관 목록 영역 |
| 계약관리 | 계약업체관리 | 계약업체 목록 영역 |
| 계약관리 | 계약문서관리 | 계약문서 PDF 영역 |
| 계약관리 | 계약변경이력관리 | 변경이력 조회 영역 |
| 계약관리 | 프로젝트 기준정보 생성 | 계약 후속업무 영역 |

모든 기능은 계약관리 화면 `CON-001`에서 제공한다. 목록에서 계약을 선택하면 동일한 계약 상세 화면 안에서 계약정보를 중심으로 계약기관·계약업체 목록, 계약문서 PDF, 변경이력 및 프로젝트 생성현황을 연속된 영역으로 조회하고 관리한다. 각 항목을 별도 업무 화면이나 독립 계약으로 취급하지 않는다.

# 3. 상세 업무기능

## 3.1 계약관리

### 3.1.1 계약정보관리

| 기능ID | 기능명 | 설명 |
| --- | --- | --- |
| BFD-05-01-01-01 | 계약 목록조회 | 계약번호, 계약건명, 고객사, 계약유형, 계약상태, 계약기간 등으로 계약을 조회한다. |
| BFD-05-01-01-02 | 계약 상세조회 | 계약 기본정보와 계약기관·계약업체·계약문서·변경이력 및 프로젝트 생성현황을 통합 조회한다. |
| BFD-05-01-01-03 | 계약 직접등록 | 고객사를 기준으로 수주정보 없이 계약 업무대상과 계약 기본정보를 하나의 처리 단위로 등록한다. |
| BFD-05-01-01-04 | 수주계약 등록 | 수주정보가 존재하는 경우 수주결과를 참조하여 계약 업무대상과 계약을 등록하고 수주와 계약의 연계관계를 유지한다. |
| BFD-05-01-01-05 | 계약정보 수정 | 계약건명, 방법, 유형, 품명, 수량 및 주요 금액·기간정보를 수정하고 변경 대상에 따라 변경이력을 생성한다. |
| BFD-05-01-01-06 | 계약상태 변경 | 작성, 진행, 완료, 해지 등 계약의 현재 상태를 변경한다. |
| BFD-05-01-01-07 | 계약 삭제 | 업무 참조가 없는 잘못 등록된 계약을 논리 삭제한다. |

### 3.1.2 계약기관관리

계약기관은 계약서에 표시되는 발주처, 계약상대자, 수요기관 등 계약 관계기관의 당시 정보를 보존한다.

| 기능ID | 기능명 | 설명 |
| --- | --- | --- |
| BFD-05-01-02-01 | 계약기관 조회 | 계약별 기관구분, 기관명, 사업자등록번호 및 담당자 정보를 조회한다. |
| BFD-05-01-02-02 | 계약기관 등록 | 기관구분과 기관 기본정보 및 계약 담당자 연락정보를 등록한다. |
| BFD-05-01-02-03 | 계약기관 수정 | 기관명, 주소, 대표자, 사업자등록번호 및 담당자 정보를 수정한다. |
| BFD-05-01-02-04 | 계약기관 삭제 | 잘못 등록되었거나 더 이상 유효하지 않은 계약기관을 논리 삭제한다. |

### 3.1.3 계약업체관리

계약업체는 공동수급 등 계약에 참여하는 업체와 업체별 지분정보를 관리한다.

| 기능ID | 기능명 | 설명 |
| --- | --- | --- |
| BFD-05-01-03-01 | 계약업체 조회 | 계약별 참여업체, 대표업체 여부, 지분율 및 지분금액을 조회한다. |
| BFD-05-01-03-02 | 계약업체 등록 | 상호명, 대표자, 사업자등록번호, 주소와 지분정보를 등록한다. |
| BFD-05-01-03-03 | 계약업체 수정 | 참여업체 기본정보, 지분율, 지분금액 및 대표업체 여부를 수정한다. |
| BFD-05-01-03-04 | 대표업체 지정 | 계약 참여업체 중 하나를 대표업체로 지정한다. |
| BFD-05-01-03-05 | 계약업체 삭제 | 잘못 등록된 계약 참여업체를 논리 삭제하고 지분 합계를 다시 검증한다. |

### 3.1.4 계약문서관리

계약문서는 계약정보·계약기관·계약업체 내용이 포함된 실제 계약서 PDF와 이후 발생한 변경계약서 PDF를 보존한다.

| 기능ID | 기능명 | 설명 |
| --- | --- | --- |
| BFD-05-01-04-01 | 계약문서 목록조회 | 현재 계약에 등록된 계약서·변경계약서 PDF의 문서유형, 문서명 및 등록정보를 조회한다. |
| BFD-05-01-04-02 | 계약문서 PDF 등록 | 계약문서 업무대상, 문서유형·문서정보와 동일 업무대상에 소속된 PDF 첨부파일을 하나의 처리 단위로 등록한다. |
| BFD-05-01-04-03 | 계약문서 PDF 조회 | 권한이 있는 사용자가 등록된 계약문서 PDF를 화면에서 확인한다. |
| BFD-05-01-04-04 | 계약문서 다운로드 | 권한이 있는 사용자가 계약문서 PDF를 다운로드한다. |
| BFD-05-01-04-05 | 계약문서 정보수정 | 문서유형과 문서명을 수정한다. |
| BFD-05-01-04-06 | 계약문서 PDF 교체 | 잘못 첨부한 PDF를 동일한 계약문서 업무대상에 소속된 새 PDF로 교체한다. 변경계약서는 기존 PDF를 교체하지 않고 별도 계약문서와 업무대상으로 등록한다. |
| BFD-05-01-04-07 | 계약문서 삭제 | 업무상 보존 대상이 아닌 잘못 등록된 계약문서를 논리 삭제한다. |

### 3.1.5 계약변경이력관리

계약변경이력은 확정된 계약의 금액, 기간, 범위 및 상태 등 주요 조건 변경을 차수별로 보존한다.

| 기능ID | 기능명 | 설명 |
| --- | --- | --- |
| BFD-05-01-05-01 | 계약변경이력 조회 | 계약별 변경차수, 변경유형, 변경일자 및 변경 전·후 내용을 조회한다. |
| BFD-05-01-05-02 | 계약변경 등록 | 변경유형과 변경일자를 입력하고 변경 전·후 계약내용을 변경차수별로 기록한다. |
| BFD-05-01-05-03 | 계약변경 상세조회 | 선택한 변경차수의 변경 대상과 변경 전·후 내용을 상세조회한다. |
| BFD-05-01-05-04 | 변경이력 자동생성 | 주요 계약정보 수정 시 수정 전 값을 보존하고 수정 후 값과 함께 변경이력을 생성한다. |

계약변경이력은 감사 목적의 이력정보이므로 등록 후 직접 수정하거나 삭제하지 않는다. 오류 정정은 다음 변경차수로 기록한다.

### 3.1.6 프로젝트 기준정보 생성

| 기능ID | 기능명 | 설명 |
| --- | --- | --- |
| BFD-05-01-06-01 | 프로젝트 생성대상 확인 | 계약상태와 기존 프로젝트 생성현황을 확인하여 프로젝트 생성 가능 여부를 판단한다. |
| BFD-05-01-06-02 | 프로젝트 기준정보 입력 | 프로젝트명, 유형, 시작일자, 종료일자, 상태 및 PM을 입력한다. |
| BFD-05-01-06-03 | 프로젝트 생성 | 프로젝트 업무대상과 계약ID·고객ID를 참조하는 프로젝트 기준정보를 하나의 처리 단위로 생성한다. |
| BFD-05-01-06-04 | 생성 프로젝트 조회 | 계약에서 생성된 프로젝트 기준정보와 인력투입 사용 여부를 조회한다. |

프로젝트 생성 후 별도 프로젝트관리 화면은 제공하지 않는다. 프로젝트 기준정보는 계약 상세에서 조회하고 인력관리·예산·원가·매출 업무에서 참조한다.

# 4. 계약관리 화면 구성

## 4.1 목록영역

| 구분 | 내용 |
| --- | --- |
| 검색조건 | 계약번호, 계약건명, 고객사, 계약유형, 계약상태, 계약 시작·종료기간 |
| 주요 표시항목 | 계약번호, 계약건명, 고객사, 계약금액, 계약기간, 계약상태, 프로젝트 생성 여부 |
| 주요 기능 | 조회, 신규등록, 상세선택 |

## 4.2 계약 상세 통합영역

| 구성영역 | 주요 내용 |
| --- | --- |
| 기본정보 | 계약번호, 고객사, 선택 수주, 계약건명, 방법, 유형, 상태, 품명, 수량, 금액, 기간, 납품기한, 세금·보증·채권 정보 |
| 계약기관 | 발주처, 계약상대자, 수요기관 및 기관 담당자 정보 |
| 계약업체 | 계약 참여업체, 대표업체, 지분율 및 지분금액 |
| 계약문서 PDF | 실제 계약서·변경계약서 PDF 등록, 조회, 다운로드 및 교체 |
| 변경이력 | 변경차수, 변경유형, 변경일자 및 변경 전·후 내용 |
| 프로젝트 | 계약에서 생성된 프로젝트 기준정보와 프로젝트 생성 기능 |

계약 상세는 하나의 계약 컨텍스트에서 모든 구성영역을 함께 표시한다. 신규 계약은 기본정보를 저장하여 계약ID를 생성한 후 같은 화면에서 계약기관·계약업체와 계약문서 PDF를 이어서 등록한다. 하위정보의 추가·수정·삭제는 영역별로 저장하되 항상 현재 계약ID에 종속하여 처리한다.

# 5. 주요 업무 규칙

| 구분 | 업무 규칙 |
| --- | --- |
| 계약 식별 | 계약번호는 삭제되지 않은 계약 간 중복을 허용하지 않는다. |
| 고객 참조 | 계약은 하나의 고객사를 참조해야 한다. |
| 수주 참조 | 수주ID는 선택값이며 수주 없이 계약을 직접 등록할 수 있다. 하나의 수주는 최대 하나의 계약과 연결한다. |
| 업무대상 생성 | 계약, 계약문서와 프로젝트를 생성할 때 각각의 업무대상과 업무 엔터티를 동일 트랜잭션에서 생성한다. |
| 계약기간 | 계약시작일자는 계약종료일자보다 늦을 수 없다. |
| 납품기한 | 납품기한이 계약기간과 다르면 사용자에게 확인 경고를 제공한다. |
| 금액 | 계약금액, 보증금액, 인지세액 및 채권금액은 음수가 될 수 없다. |
| 과세 | 과세대상 여부에 따른 세금 관련 처리 기준은 후속 설계에서 확정한다. |
| 계약기관 | 동일 계약에 같은 기관구분과 사업자등록번호의 기관을 중복 등록하지 않는다. |
| 계약업체 | 동일 계약에 같은 사업자등록번호의 업체를 중복 등록하지 않는다. |
| 대표업체 | 공동수급 계약은 대표업체를 최대 한 곳만 지정한다. |
| 지분 | 지분을 관리하는 계약은 참여업체 지분율 합계를 100%로 검증하고 지분금액 합계를 계약금액과 비교한다. |
| 변경차수 | 같은 계약에서 변경차수는 중복될 수 없으며 순차적으로 증가한다. |
| 변경이력 | 주요 계약조건 변경 전·후 값을 보존하며 기존 변경이력은 수정·삭제하지 않는다. |
| 계약문서 | 하나의 계약문서 레코드는 하나의 공통 첨부파일을 필수·고유 참조하고 계약문서와 첨부파일의 업무대상ID가 일치해야 한다. |
| 문서형식 | 계약문서 첨부는 PDF 형식만 허용하며 원본 계약서와 변경계약서는 별도 계약문서로 보존한다. |
| 프로젝트 | 하나의 계약에서 하나 이상의 프로젝트를 생성할 수 있으며 프로젝트 기간은 원칙적으로 계약기간 안에 있어야 한다. |
| 삭제정책 | 프로젝트, 계약문서 또는 후속 업무에서 참조 중인 계약은 삭제하지 않고 계약상태를 변경한다. |

# 7. 예외 및 오류처리 기준

| 상황 | 처리 기준 |
| --- | --- |
| 동일 계약번호 등록 | 저장을 차단하고 기존 계약을 안내한다. |
| 삭제되거나 사용할 수 없는 고객 선택 | 계약 저장을 차단하고 유효한 고객 선택을 안내한다. |
| 이미 계약과 연결된 수주 선택 | 중복 계약 생성을 차단하고 기존 계약을 안내한다. |
| 계약 시작일이 종료일보다 늦음 | 저장을 차단하고 계약기간 수정을 안내한다. |
| 계약업체 지분율 합계 불일치 | 지분관리 대상 계약은 저장 또는 확정을 차단하고 차이를 안내한다. |
| 대표업체 복수 지정 | 기존 대표업체를 안내하고 하나의 대표업체만 선택하도록 한다. |
| 참조 중인 계약 삭제 | 삭제를 차단하고 완료·해지·취소 등 상태 변경을 안내한다. |
| 계약문서 파일 처리 실패 | 계약 기본정보와 다른 하위정보는 유지하고 실패한 문서만 재등록할 수 있게 한다. |
| PDF가 아닌 계약문서 등록 | 파일 등록을 차단하고 PDF 형식의 실제 계약문서를 첨부하도록 안내한다. |
| 계약문서와 PDF 업무대상 불일치 | 계약문서 등록·교체를 차단하고 동일한 계약문서 업무대상에 PDF를 다시 등록하도록 안내한다. |
| 프로젝트 중복 생성 | 기존 생성 프로젝트를 안내하고 추가 프로젝트 생성 여부를 확인한다. |
| 계약기간 밖의 프로젝트 생성 | 경고 후 정책에 따라 저장을 제한하거나 별도 확인을 받는다. |

## 8.2 주요 관계

| 관계 | 설명 |
| --- | --- |
| 고객사 1 : N 계약 | 하나의 고객사와 여러 계약을 체결할 수 있다. |
| 수주 1 : 0..1 계약 | 수주에서 계약이 생성될 수 있으며 직접 등록 계약은 수주를 참조하지 않는다. |
| 계약 1 : N 계약기관 | 계약은 발주처·계약상대자·수요기관 등 여러 기관정보를 가진다. |
| 계약 1 : N 계약업체 | 계약은 여러 참여업체와 지분정보를 가질 수 있다. |
| 계약 1 : N 계약변경이력 | 계약에는 여러 차수의 변경이력이 발생할 수 있다. |
| 계약 1 : N 계약문서 | 계약에는 여러 계약 관련 문서를 첨부할 수 있다. |
| 계약 1 : 0..N 프로젝트 | 계약 등록 시 프로젝트가 없을 수 있고 이후 하나 이상의 프로젝트 기준정보를 생성할 수 있다. |
| 업무대상 1 : 0..1 계약 | 계약은 업무대상ID를 필수·고유 참조한다. |
| 업무대상 1 : 0..1 계약문서 | 계약문서는 계약과 별개의 업무대상ID를 필수·고유 참조한다. |
| 업무대상 1 : 0..1 프로젝트 | 프로젝트는 업무대상ID를 필수·고유 참조한다. |
| 첨부파일 1 : 0..1 계약문서 | 모든 계약문서는 동일 업무대상에 소속된 PDF 파일 하나를 필수·고유 참조한다. |

# 10. 후속 검토 사항

- 계약 확정 및 변경계약 승인 절차를 적용할지 검토한다.
- 변경 전·후 내용을 항목별 이력으로 저장할지 JSON 등 스냅샷으로 저장할지 확정한다.
- 계약기관의 기관구분별 필수 건수와 중복 허용 기준을 확정한다.

- 계약 하나를 여러 프로젝트로 나누는 기준과 프로젝트명 중복 기준을 확정한다.
- 계약기간을 벗어난 프로젝트 기간 허용 및 승인 기준을 확정한다.
