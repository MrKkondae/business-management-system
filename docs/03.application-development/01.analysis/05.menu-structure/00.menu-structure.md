# 메뉴구조도

## 1. 문서 개요

### 1.1 목적

본 문서는 BMS(Business Management System)의 메뉴 구성을 정의한다.

메뉴구조도는 업무기능분해도 및 상세업무기능분해도를 기반으로 사용자에게 제공되는 메뉴를 정의하며, 화면목록 및 권한관리의 기준으로 활용한다.

---

### 1.2 메뉴 구성 원칙

* 메뉴는 업무영역을 기준으로 구성한다.
* 하나의 메뉴는 하나의 화면을 기준으로 한다.
* 조회, 등록, 수정, 삭제 기능은 하나의 관리 화면에서 제공하는 것을 원칙으로 한다.
* 메뉴 권한은 역할(Role) 기반으로 관리한다.

# 3. 메뉴 구현단계

메뉴는 삭제하지 않고 구현단계를 다음과 같이 구분한다.

| 구현단계 | 적용 기준 |
| --- | --- |
| 1차 | 현재 상세기능과 데이터 모델을 기준으로 화면설계 및 구현을 진행한다. |
| 후속 | 메뉴와 화면 ID는 예약하되 1차 화면설계·개발·운영 메뉴 노출 대상에서 제외한다. |
| 제외 | 구현범위에서 제외하며 메뉴와 화면 ID를 생성하지 않는다. |

| 구현단계 | 1Depth | 2Depth | 비고 |
| --- | --- | --- | --- |
| 1차 | 대시보드 | 일반 사용자 대시보드 | MVP 포함 |
