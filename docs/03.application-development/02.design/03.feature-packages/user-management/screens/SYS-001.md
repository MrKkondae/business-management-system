# SYS-001 사용자관리 화면 설계 초안

## 1. 화면 정보

| 항목 | 내용 |
| --- | --- |
| 화면 ID | `SYS-001` |
| URL | `/system/users` |
| 기능 패키지 | `FPK-SYS-USER-001` |
| 권한 | `SYS-001` 메뉴 권한, 기본 역할 시스템관리자 |
| 이번 단위 | 사용자 목록과 신규 등록 |

## 2. 화면 구성

```text
[검색어................] [계정상태▼] [조직▼] [역할▼] [조회] [초기화] [사용자 등록]

사원번호 | 사용자명 | 로그인ID | 조직 | 역할 | 이메일 | 휴대전화 | 계정상태 | 마지막로그인
---------------------------------------------------------------------------------------
...

[이전] 1 2 3 [다음]                                  [20▼]
```

`사용자 등록`을 선택하면 등록 모달 또는 동일 URL의 패널을 연다. 상세·수정 화면 이동은 후속 단위에서 추가한다.

## 3. 조회조건과 목록

| 항목 | 기준 |
| --- | --- |
| 검색어 | 로그인ID·사용자명·사원번호, 앞뒤 공백 제거 |
| 계정상태 | 전체, 활성, 비활성 |
| 조직 | 미삭제 조직 옵션 |
| 역할 | 미삭제 역할 옵션 |
| 페이지 크기 | 20, 50, 100 |
| 기본 정렬 | 등록일시 내림차순 |

조회조건, 페이지와 정렬은 URL query parameter에 보존한다. 이메일과 휴대전화는 항상 마스킹한다.

## 4. 등록 입력

| 필드 | 필수 | 입력 기준 |
| --- | :---: | --- |
| 사원번호 | Y | 1~30자, 앞뒤 공백 제거 |
| 성명 | Y | 1~100자, 앞뒤 공백 제거 |
| 조직 | Y | 미삭제 조직 중 하나 |
| 로그인ID | Y | 4~26자, 영문자·숫자·점·밑줄·하이픈, 첫 글자는 영문자 또는 숫자; 저장 시 소문자 변환 |
| 초기 역할 | Y | 미삭제 역할 한 건 이상, 복수 선택 가능 |
| 이메일 | N | 최대 100자, 이메일 형식 |
| 휴대전화 | N | 최대 20자, 숫자와 하이픈 허용 |

로그인ID는 입력 중 대문자를 소문자로 변환하고 저장값을 사용자에게 명확히 보여준다. 외주인력 여부, 상세 경력, 직위와 입사일자는 이 등록 폼에서 입력하지 않는다.

## 5. 등록 처리와 결과

1. 필수·형식 오류를 확인하고 첫 오류 필드로 포커스를 이동한다.
2. 저장 중 입력과 등록 버튼을 비활성화한다.
3. 성공하면 24시간 유효한 최초 등록 전용 임시 비밀번호를 한 번만 보여주는 완료 대화상자를 표시한다.
4. 대화상자는 로그인ID, 임시 비밀번호, 만료일시와 안전한 전달 안내를 표시한다.
5. 임시 비밀번호 복사 버튼은 제공할 수 있으나 클립보드 자동 복사는 하지 않는다.
6. 완료 대화상자를 닫으면 임시 비밀번호를 다시 조회할 수 없다는 경고 후 목록을 갱신한다.

임시 비밀번호를 URL, 브라우저 저장소, 분석 이벤트와 콘솔 로그에 기록하지 않는다.

## 6. 화면 상태

| 상태 | 처리 |
| --- | --- |
| 최초·조회 중 | 기존 목록을 유지하고 진행 표시 |
| 결과 없음 | “조회 결과가 없습니다”와 조건 초기화 안내 |
| 검증 실패 | 필드 인접 오류와 첫 오류 포커스 |
| 로그인ID 중복 | 로그인ID 필드에 중복 메시지 |
| 사원번호 중복 | 사원번호 필드에 중복 메시지 |
| 조직·역할 상태 변경 | 옵션을 재조회하고 다시 저장하도록 안내 |
| 인증 만료 | 로그인 화면으로 이동 |
| 권한 없음 | 403 안내 화면 표시 |
| 서버 오류 | `traceId`와 재시도 안내 표시 |

## 7. API 연결

| API ID | 용도 |
| --- | --- |
| `API-SYS-USER-001` | 사용자 목록 조회 |
| `API-SYS-USER-003` | 조직·역할 등록 옵션 조회 |
| `API-SYS-USER-002` | 직원·사용자·초기 역할 등록 |

Method·Path와 권한·오류 연결은 `../../../02.catalog/apis.csv`를 원본으로 한다.

## 8. 화면 인수조건

- 조회조건과 페이지가 새로고침 후 유지된다.
- 이메일과 휴대전화 원문이 목록에 표시되지 않는다.
- 등록 중 중복 제출이 차단된다.
- 필드별 중복 오류가 해당 입력에 표시된다.
- 성공 응답의 임시 비밀번호는 완료 대화상자에서 한 번만 확인할 수 있다.
- 키보드만으로 조회, 등록, 오류 확인과 완료 대화상자 닫기가 가능하다.
