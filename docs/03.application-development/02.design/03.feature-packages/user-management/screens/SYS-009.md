# SYS-009 최초 등록 화면 상세설계

## 1. 화면 정보

| 항목 | 내용 |
| --- | --- |
| 설계 상태 | `READY` |
| 화면 ID | `SYS-009` |
| URL | `/account/initial-registration` |
| 기능 패키지 | `FPK-SYS-USER-001` |
| 관련 기능 | `BFD-01-01-02-02` |
| 접근 조건 | `passwordChangeRequired=true`인 유효한 제한 세션 |
| 제외 범위 | 일반 비밀번호 변경, 비밀번호 찾기, 관리자 비밀번호 초기화 |

## 2. 화면 목적

초기 비밀번호로 로그인한 사용자가 새 비밀번호와 선택 연락처를 등록하여 제한 세션을 일반 세션으로 승격한다. 완료 전에는 업무 메뉴를 표시하지 않고 허용된 인증·최초 등록 API만 호출한다.

## 3. 화면 구성

### 데스크톱

```text
┌──────────────────────────────────────────────────────────────┐
│ BMS                                               [로그아웃] │
├──────────────────────────────────────────────────────────────┤
│ 계정 설정을 완료해 주세요                                   │
│ 초기 비밀번호를 변경한 후 업무 메뉴를 사용할 수 있습니다.  │
│                                                              │
│ 새 비밀번호            [••••••••••••             ] [표시]   │
│ 비밀번호 확인          [••••••••••••             ] [표시]   │
│ 이메일(선택)           [                             ]        │
│ 휴대전화(선택)         [                             ]        │
│                                                              │
│ 비밀번호 조건                                                │
│ · 12~64자 · 문자종류 3종 이상 · 개인정보/취약문자열 제외   │
│                                                              │
│ [오류 또는 처리 안내]                         [등록 완료]    │
└──────────────────────────────────────────────────────────────┘
```

### 모바일

- 한 열로 배치하고 본문 좌우 여백은 최소 20px을 유지한다.
- 버튼과 입력 높이는 최소 44px로 한다.
- 비밀번호 조건과 오류는 입력 영역 다음에 표시한다.
- 가로 스크롤을 만들지 않는다.

## 4. 진입과 이탈

1. 화면 진입 시 `GET /auth/me`로 세션을 확인한다.
2. `401`이면 `/login`으로 이동한다.
3. 일반 세션이면 `/`로 이동한다.
4. 제한 세션이면 사용자 표시명과 절대 만료일시를 메모리에 두고 폼을 표시한다.
5. 사용자가 URL을 직접 변경해도 인증 진입점에서 제한 세션을 다시 이 화면으로 보낸다.
6. 로그아웃 성공 또는 세션 만료 시 민감 입력을 비우고 `/login`으로 이동한다.

## 5. 입력 필드

| 필드 | HTML/자동완성 | 필수 | 클라이언트 검증 | 전송값 |
| --- | --- | :---: | --- | --- |
| 새 비밀번호 | `password`, `new-password` | Y | 12~64자, 아래 정책 | 원문 |
| 새 비밀번호 확인 | `password`, `new-password` | Y | 새 비밀번호와 일치 | 원문 |
| 이메일 주소 | `email`, `email` | N | 최대 100자, 입력 시 이메일 형식 | 앞뒤 공백 제거, 빈 값은 `null` |
| 휴대전화 번호 | `tel`, `tel` | N | 최대 20자, 숫자·`+()-`·공백만 허용 | 앞뒤 공백 제거, 빈 값은 `null` |

비밀번호 정책은 백엔드 `NewPasswordPolicy`와 동일하게 다음을 적용한다.

- 12자 이상 64자 이하
- 대문자, 소문자, 숫자, 특수문자 중 3종류 이상
- 로그인ID 또는 표시명이 3자 이상이면 해당 값을 포함하지 않음
- `password`, `qwerty`, `letmein`, `welcome`, `admin`을 대소문자 구분 없이 포함하지 않음
- 오름차순·내림차순 4문자 연속, 동일 문자 4회 연속 및 전체 반복 패턴 제외

클라이언트 검증은 사용자 피드백을 위한 것이며 서버 검증을 대신하지 않는다.

## 6. 버튼과 동작

| 동작 | 처리 |
| --- | --- |
| 비밀번호 표시 | 각 필드 값을 독립적으로 표시·숨김 처리하고 접근성 이름을 “비밀번호 표시/숨기기”로 변경 |
| 등록 완료 | 클라이언트 검증 후 CSRF 토큰을 첨부해 `API-SYS-USER-004` 호출 |
| Enter | 폼 제출과 동일하게 처리하되 제출 중에는 무시 |
| 로그아웃 | 확인 대화상자 없이 `API-COM-AUTH-002` 호출 후 `/login` 이동 |

등록 성공 시:

1. 비밀번호 값을 즉시 비운다.
2. 기존 CSRF 토큰을 폐기한다.
3. `GET /auth/me`를 다시 호출해 `passwordChangeRequired=false`를 확인한다.
4. 구현된 허용 메뉴가 있으면 첫 메뉴로, 없으면 `/`의 공통 애플리케이션 기본 화면으로 이동한다.

## 7. 화면 상태와 오류

| 상태/오류 | 표시와 동작 |
| --- | --- |
| 세션 확인 중 | 폼 대신 로딩 상태를 표시하며 입력 포커스를 이동하지 않음 |
| 초기 | 모든 입력은 빈 값, 등록 완료 버튼 활성 |
| 제출 중 | 모든 입력과 버튼을 비활성화하고 중복 요청 차단 |
| 클라이언트 검증 실패 | 필드 인접 메시지, 요약 알림, 첫 오류 필드 포커스 |
| `COMMON_VALIDATION_FAILED` | `fieldErrors`를 해당 필드에 연결하고 연결되지 않은 오류는 폼 상단 표시 |
| `COMMON_INVALID_STATE` | “현재 계정 상태가 변경되었습니다” 안내 후 `/auth/me` 재조회 |
| `AUTH_AUTHENTICATION_REQUIRED` | 값을 비우고 `/login` 이동 |
| `AUTH_CSRF_INVALID` | 토큰을 한 번 갱신해 동일 제출을 한 번만 재시도 |
| `429` | 현재 API 계약에는 없으므로 일반 오류로 처리하지 않으며 계약 변경 전 임의 재시도 금지 |
| 네트워크·서버 오류 | 안전한 메시지와 서버가 제공한 경우 `traceId`, 재시도 버튼 표시 |

서버 오류 메시지에 비밀번호 정책의 어느 항목이 실패했는지 세분화되어 있지 않으므로, 클라이언트는 제출 전에 정책 항목을 모두 확인해 구체적으로 안내한다.

## 8. API 연결

| 순서 | API ID | 목적 |
| ---: | --- | --- |
| 1 | `API-COM-AUTH-003` | 제한 세션 및 현재 사용자 확인 |
| 2 | `API-COM-AUTH-005` | 상태 변경용 CSRF 토큰 준비 |
| 3 | `API-SYS-USER-004` | 최초 등록과 초기 비밀번호 변경 |
| 조건부 | `API-COM-AUTH-004` | 실제 사용자 활동 갱신 |
| 조건부 | `API-COM-AUTH-002` | 로그아웃 |

요청 본문:

```json
{
  "newPassword": "new-password-value",
  "newPasswordConfirmation": "new-password-value",
  "emailAddress": null,
  "mobileNumber": null
}
```

## 9. 접근성과 보안

- 페이지 제목과 첫 제목은 “최초 등록” 의미를 제공한다.
- 모든 입력에 화면에 보이는 레이블을 연결한다.
- 오류는 `aria-describedby`, 폼 요약은 `role="alert"`로 연결한다.
- 제출 후 오류가 있으면 첫 오류 필드, 서버 공통 오류면 오류 요약으로 포커스를 이동한다.
- 비밀번호 조건은 색상만으로 충족 여부를 표현하지 않는다.
- 비밀번호를 전역 상태, 브라우저 저장소, URL, 로그와 분석 이벤트에 저장하지 않는다.
- 뒤로 가기와 새로고침 후 비밀번호 필드를 복원하지 않는다.
- 제한 세션에서는 역할과 업무 메뉴를 표시하지 않는다.

## 10. 인수조건

- 제한 세션만 화면을 사용할 수 있고 일반 세션과 미인증 사용자는 지정 경로로 이동한다.
- 필수값, 길이, 문자종류, 금지 문자열, 연속·반복 패턴과 확인값 불일치를 API 호출 전에 안내한다.
- 선택 연락처의 빈 값은 `null`로 전송한다.
- 제출 중 클릭과 Enter가 중복 요청을 만들지 않는다.
- 서버 필드 오류는 해당 입력과 연결된다.
- 성공 후 세션ID가 교체되고 `/auth/me`에서 일반 세션이 확인된 뒤 이동한다.
- 성공·실패·로그아웃 시 비밀번호가 메모리와 화면에서 제거된다.
- 키보드만으로 입력, 비밀번호 표시, 제출, 오류 확인과 로그아웃을 수행할 수 있다.
