# COM-001 로그인 화면 설계 초안

## 1. 화면 정보

| 항목 | 내용 |
| --- | --- |
| 화면 ID | `COM-001` |
| URL | `/login` |
| 기능 패키지 | `FPK-COM-AUTH-001` |
| 인증 | 비인증 사용자 접근 가능 |
| 관련 기능 | `BFD-02-05-01-01`, `BFD-02-05-01-02`, `BFD-02-05-01-03` |

## 2. 화면 목적

로그인ID와 비밀번호를 입력받아 서버 세션을 생성한다. 이미 인증된 일반 세션으로 접근하면 접근 가능한 첫 업무 화면으로 이동한다.

## 3. 입력 필드

| 필드 | 타입 | 필수 | 제약과 표시 |
| --- | --- | :---: | --- |
| 로그인ID | text | Y | 앞뒤 공백 제거 후 영문자를 소문자로 변환, 자동완성 `username`, 레이블 제공 |
| 비밀번호 | password | Y | 자동완성 `current-password`, 화면과 로그에 원문 재표시 금지 |

비밀번호 표시 전환 기능은 v1.0 초안에 포함하지 않는다. 추가할 경우 버튼의 접근성 이름과 재숨김 동작을 함께 정의한다.

## 4. 동작

| 동작 | 처리 |
| --- | --- |
| 화면 진입 | CSRF 토큰을 준비하고 로그인ID 입력에 포커스한다. |
| 로그인 | 중복 제출을 차단하고 `POST /auth/login`을 호출한다. |
| Enter 입력 | 비밀번호 필드에서 로그인과 동일하게 처리한다. |
| 로그인 성공 | 일반 세션이면 현재 구현되어 있고 접근 가능한 첫 업무 메뉴로 이동한다. 접근 가능한 구현 메뉴가 없으면 권한 안내 화면으로 이동한다. |
| 초기 비밀번호 사용자 | 사용자 관리 패키지가 소유하는 `/account/initial-registration`으로 이동한다. |
| 로그인 실패 | 로그인ID 존재 여부와 무관하게 “로그인ID 또는 비밀번호를 확인해 주세요.”를 표시한다. |
| 속도 제한 | “로그인 요청이 많습니다. 잠시 후 다시 시도해 주세요.”를 표시하고 `Retry-After` 동안 제출을 비활성화한다. |

## 5. 화면 상태

| 상태 | 표현과 동작 |
| --- | --- |
| 초기 | 빈 비밀번호, 로그인 버튼 활성 |
| 제출 중 | 로그인 버튼과 입력을 비활성화하고 진행 상태 표시 |
| 필수값 오류 | 필드 인접 오류를 표시하고 첫 오류 필드로 포커스 이동 |
| 인증 실패 | 공통 메시지를 폼 상단에 표시하되 비밀번호를 비운다. |
| 요청 제한 | 남은 대기시간을 안내하고 시간이 지나면 다시 제출할 수 있게 한다. |
| 서버 오류 | 안전한 메시지와 `traceId`, 재시도 안내 표시 |
| 이미 로그인 | 현재 사용자 조회 후 업무 화면으로 이동 |

계정 비활성화 여부, 남은 실패 횟수와 로그인ID 존재 여부는 화면에 표시하지 않는다.

## 6. API 연결

| 순서 | API ID | 목적 |
| ---: | --- | --- |
| 1 | `API-COM-AUTH-005` | 로그인 요청용 CSRF 토큰 준비 |
| 2 | `API-COM-AUTH-001` | 인증과 세션 생성 |
| 조건부 | `API-COM-AUTH-003` | 기존 세션 확인 |

Method·Path와 권한·오류 연결은 `../../../02.catalog/apis.csv`를 원본으로 한다.

## 7. 접근성과 보안

- 입력 필드와 오류 메시지를 접근성 속성으로 연결한다.
- 인증 실패 메시지는 스크린리더가 인지할 수 있는 알림 영역에 표시한다.
- 로그인ID를 URL query, 브라우저 저장소 또는 분석 이벤트에 기록하지 않는다.
- 비밀번호를 컴포넌트 상태 외부, 전역 상태와 브라우저 저장소에 보관하지 않는다.
- 세션 쿠키와 CSRF 쿠키의 값을 애플리케이션 로그에 출력하지 않는다.

## 8. 화면 인수조건

- 필수값 없이 제출하면 API를 호출하지 않고 필드 오류가 표시된다.
- 제출 중 추가 클릭과 Enter 입력이 중복 로그인 요청을 만들지 않는다.
- 인증 실패 원인과 관계없이 동일한 메시지가 표시되고 비밀번호가 지워진다.
- 성공 시 서버가 반환한 메뉴 중 현재 구현되어 있고 접근 가능한 첫 업무 메뉴로 이동하며, 해당 메뉴가 없으면 권한 안내 화면으로 이동한다.
- 키보드만으로 입력, 제출과 오류 확인이 가능하다.
