# 인증 선행 데이터 관리 규칙

## 1. 목적

인증과 권한 기능이 동작하기 전에 필요한 역할, 메뉴, 역할-메뉴 관계, 공통코드를 재현 가능한 초기 데이터로 관리한다.

CSV는 설계 원본이며 애플리케이션이 실행 중 직접 읽는 파일이 아니다. 배포 전 검증을 통과한 CSV로 Flyway 마이그레이션 SQL을 생성하고, 운영 반영은 Flyway만 수행한다.

## 2. 설계 원본

| 파일 | 관리 대상 |
|---|---|
| `roles.csv` | 시스템 기본 역할 |
| `menus.csv` | 최초 운영 메뉴 |
| `role-menu-permissions.csv` | 역할별 메뉴 권한 |
| `code-groups.csv` | 인증·부트스트랩 공통코드 그룹 |
| `codes.csv` | 인증·부트스트랩 공통코드 |

역할명, 메뉴명, 코드값을 다른 문서에 다시 목록으로 관리하지 않는다. 다른 문서는 이 디렉터리를 참조한다.

## 3. 반영 절차

1. CSV를 변경한다.
2. `python scripts/validate_seed_data.py`로 형식, 참조 무결성, 중복, 시스템관리자 전체 메뉴 권한을 검증한다.
3. `python scripts/generate_seed_migration.py --output <신규 마이그레이션 경로>`로 SQL을 생성한다.
4. 생성 SQL을 검토하고 신규 버전 파일로 저장한다.
5. 빈 DB와 기존 버전 DB에서 각각 마이그레이션을 검증한다.
6. 배포 후 부트스트랩 명령으로 최초 조직과 최초 시스템관리자 사용자를 생성한다.

이미 적용된 Flyway 파일은 수정하지 않는다. 초기 데이터 변경도 항상 신규 마이그레이션으로 반영한다.

## 4. 시스템관리자 전체 메뉴 규칙

- 시스템관리자 역할 ID는 `01KY3HYG000000000000000001`로 고정한다.
- `menus.csv`에 등록된 모든 유효 메뉴는 `role-menu-permissions.csv`에 시스템관리자 권한이 반드시 있어야 한다.
- 상위 메뉴도 화면 메뉴와 동일하게 권한 관계를 명시한다.
- 새 메뉴를 운영에 추가할 때 메뉴 INSERT와 시스템관리자 권한 INSERT를 같은 Flyway 마이그레이션에 포함한다.
- 시스템관리자를 권한 검사에서 우회시키지 않는다. 일반 사용자와 동일한 RBAC 관계를 사용한다.
- 삭제 또는 비활성 메뉴는 메뉴 조회 단계에서 제외한다.

현재 초기 메뉴는 시스템관리 영역의 사용자, 역할, 메뉴, 공통코드, 조직 관리이다. 대시보드와 아직 범위가 확정되지 않은 업무 메뉴는 해당 기능을 구현할 때 추가한다. 따라서 “전체 메뉴”는 현재 운영 DB에 등록된 모든 유효 메뉴를 뜻한다.

## 5. 최초 역할 부여 원칙

- 최초 부트스트랩 사용자는 시스템관리자 역할만 부여한다.
- 나머지 기본 역할은 사용자·조직·업무 권한 설계에 따라 관리자가 이후 부여한다.
- 시스템관리자 외 역할의 메뉴 권한은 각 기능 설계 승인 후 신규 마이그레이션으로 추가한다.

## 6. 코드 원칙

- DB에는 코드값만 저장하고 표시명은 공통코드에서 조회한다.
- 코드값은 영문 대문자와 밑줄을 사용하며 각 컬럼 길이 제한을 넘지 않는다.
- 로그인 실패 사유는 `BAD_CREDENTIALS`로 통일하여 아이디 존재 여부를 외부에 노출하지 않는다.
- 임시 비밀번호 만료 내부 사유는 컬럼 길이를 고려해 `TEMP_PWD_EXPIRED`를 사용한다.
- 최초 최상위 조직 유형은 `COMPANY`로 고정한다.

## 7. 금지 사항

- 운영자가 사용자, 조직, 권한 관계를 SQL로 직접 입력하지 않는다.
- CSV를 애플리케이션 시작 시마다 자동 반영하지 않는다.
- 비밀번호 원문이나 해시를 CSV, Git, Flyway SQL에 저장하지 않는다.
- 기존 마이그레이션 파일을 수정하여 초기 데이터를 바꾸지 않는다.
