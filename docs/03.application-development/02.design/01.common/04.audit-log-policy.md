# 감사·로그 공통 규칙

## 1. 로그 구분과 보유

| 구분 | 목적 | 기본 보유기간 |
| --- | --- | --- |
| 접속기록 | 개인정보처리시스템 접속·조회·변경 추적 | 1년; 아래 법정 조건이면 2년 |
| 업무 감사로그 | 사용자·권한·상태·개인정보 처리 등 중요 변경 증적 | 3년 |
| 시스템 진단로그 | 장애 분석과 성능·보안 탐지 | 90일 |
| 외부연계 로그 | 호출 결과, 재시도와 정합성 추적 | 1년 |

법령, 계약 또는 회사 보안정책이 더 긴 기간을 요구하면 그 근거와 대상 필드를 등록하고 별도 보관한다. 기간 종료 후 복구할 수 없게 삭제하며 운영 백업의 만료 주기에도 반영한다.

개인정보의 안전성 확보조치 기준에 따라 5만명 이상 정보주체를 처리하거나 고유식별정보·민감정보를 처리하는 개인정보처리시스템 등은 접속기록을 2년 이상 보관한다. 개인정보 접근권한의 부여·변경·말소 내역은 3년 이상 보관한다. 시스템 범위가 이 조건에 해당하는지 운영 전 개인정보 보호책임자가 확인한다.

## 2. 공통 필드

- UTC 발생시각, 로그유형, 이벤트코드, 결과, `traceId`
- 인증된 사용자ID 또는 시스템 주체, 대상 유형과 대상ID
- 서버가 신뢰한 접속 IP, HTTP method와 정규화한 경로, 상태코드와 처리시간
- 개인정보 조회·다운로드는 조회한 정보주체 또는 대상 범위와 처리 목적

비밀번호 원문·해시, 세션ID, 쿠키, Authorization·CSRF 토큰, 서비스키, 주민등록번호, 연락처·이메일·주소의 원문, 요청·응답 본문 전체를 기록하지 않는다. URL query와 헤더는 허용 목록만 기록한다.

## 3. 필수 감사 사건

- 로그인 성공·실패·로그아웃·자동 만료와 계정 활성·비활성
- 사용자, 역할, 메뉴권한과 개인정보 접근권한 변경
- 프로젝트 상태와 기간 변경, 인력 투입·단가·정산 변경
- 개인정보 전체보기, 검색, 대량조회, 다운로드와 파기
- 마이그레이션 적용, 기준 데이터 변경, 외부연계 재처리

업무 변경과 성공 감사로그는 같은 트랜잭션으로 저장한다. 진단로그 장애가 업무 변경의 증적을 누락시키지 않도록 감사 저장소의 실패를 감지하고 운영 경보를 발생시킨다. 로그 조회와 다운로드는 시스템관리자 중 승인된 역할로 제한하고 조회 행위도 기록한다.
