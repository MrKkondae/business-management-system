# 개인정보 처리 기준

## 1. 기본 원칙

- 처리 목적에 필요한 최소 항목만 수집하고 목적·근거·기산일·보유기간·파기방법을 개인정보 카탈로그에 기록한다.
- 보유기간 경과 또는 목적 달성 시 지체 없이 파기한다. 다른 법령에 따른 보존분은 운영 데이터와 분리하고 해당 목적에만 사용한다.
- 논리삭제는 업무 화면에서의 사용 중단 수단이지 법적 파기의 완료로 보지 않는다.
- 본 문서의 기간은 BMS 기본 운영안이다. 1년 운영 보유를 적용하려면 개인정보 보호법 제15조의 처리 근거와 구체적 목적을 개인정보 처리방침·내부관리계획에 기록해야 한다. 해당 근거 또는 계속되는 목적이 없으면 퇴직·계약 종료 시 운영용 개인정보를 지체 없이 파기한다.
- 개인정보 보호책임자가 실제 근로·외주계약, 세무 처리와 회사 개인정보 처리방침을 확인한 뒤 운영 적용한다.

## 2. 분류와 취급

| 등급 | 예시 | 기본 처리 |
| --- | --- | --- |
| 인증비밀 | 비밀번호, 세션·CSRF 토큰 | 화면·응답·로그 금지, 비밀번호는 복호화 불가 해시 |

| 인력 개인정보 | 성명, 생년월일, 성별, 휴대전화, 이메일, 주소 | 인력관리 승인 역할만 조회, 목록 마스킹 |
| 직무·경력정보 | 사번, 조직, 직급, 학력, 경력, 자격, 기술 | 업무 필요 범위에서 조회, 외부 제공·다운로드 제한 |

| 접속정보 | 사용자ID, IP, 사용자 에이전트, 접속시각 | 보안·감사 목적으로만 사용 |

마스킹 기본값은 휴대전화 `010-****-1234`, 이메일 `ab***@domain.com`, 생년월일 `YYYY-**-**`, 성명은 일반 업무 목록에서 가운데 글자 마스킹이다. 상세 원문 조회와 다운로드는 별도 권한으로 분리하고 사유 입력과 감사로그를 남긴다. v1.0에서 개인정보 일괄 다운로드는 기본 차단한다.

## 3. 인력정보 보유 기준

### 3.1 자사직원

| 데이터 묶음 | 기산일과 기간 | 기간 종료 처리 |
| --- | --- | --- |
| 운영용 인력 프로필·연락처 | 퇴직일 다음 날부터 1년 | 휴대전화·개인이메일·주소·생년월일·성별 파기, 이름은 아래 증빙과 분리 |
| 근로자 명부·근로계약 관련 중요 서류 | 근로관계 종료일부터 3년 | 법정기간 종료 후 다른 근거가 없으면 파기 |
| 급여·비용·세무 증빙 | 해당 국세 법정신고기한 다음 날부터 5년 | 증빙 목적의 최소 항목만 분리 보관 후 파기 |
| 프로젝트 수행 이력 | 운영 프로필 종료 후 | 통계·실적 목적이면 인력ID를 비식별 대체키로 전환하고 연락 가능한 식별정보 제거 |

“퇴직 후 1년”은 재입사 처리, 경력 확인과 진행 중 업무 인수인계의 목적과 적법한 처리 근거가 문서화된 경우의 상한 기본안이다. 그 목적이 먼저 끝나면 1년을 기다리지 않고 파기한다. 근로기준법상 3년 보존 대상까지 1년에 삭제하지 않으며, 반대로 법정 증빙을 이유로 연락처·주소 등 모든 프로필을 3년 또는 5년 보관하지 않는다.

### 3.2 외주인력

외주인력의 종료기준일은 `마지막 프로젝트 투입 종료일`, `해당 외주계약 종료일`, `관련 프로젝트의 완료·취소일` 중 가장 늦은 날로 한다. 프로젝트가 연장되면 현재 종료일이 아니라 실제 마지막 투입·계약 종료를 포함해 다시 계산한다.

| 데이터 묶음 | 기산일과 기간 | 기간 종료 처리 |
| --- | --- | --- |
| 운영용 프로필·연락처·기술정보 | 종료기준일 다음 날부터 1년 | 연락처·개인이메일·생년월일·성별 파기, 재투입 후보 기능에서 제외 |
| 외주계약·검수·지급·세무 증빙 | 해당 국세 법정신고기한 다음 날부터 5년 | 계약·정산 증빙의 최소 식별정보를 분리 보관 후 파기 |
| 프로젝트 수행 이력 | 운영 프로필 종료 후 | 프로젝트 실적 보존이 필요하면 비식별 대체키로 전환 |

외주인력이 실질적으로 회사와 근로계약을 체결한 근로자에 해당하는 경우에는 명칭과 관계없이 자사직원의 근로관계 보존 기준을 적용한다. 협력업체가 보유한 인력정보를 제공받는 경우 계약서에 제공 목적, 항목, 보유기간, 파기와 정보주체 고지 책임을 명시한다.

외주인력의 1년 운영 보유도 재투입 후보 관리 또는 계약상 하자·분쟁 대응 등 구체적 목적과 처리 근거가 있는 경우에만 적용한다. 단순히 향후 필요할 수 있다는 이유만으로 연락처와 상세 프로필을 보유하지 않는다.

## 4. 계정과 다른 개인정보

- 내부 직원 계정은 퇴직 처리 즉시 비활성화하고 모든 세션을 종료한다. 계정 식별자와 권한 변경 감사증적은 감사로그 기간 동안 분리 보관한다.
- 고객·협력업체 담당자 연락처는 마지막 거래·접촉 또는 계약 종료 후 1년을 기본으로 하되 진행 중 계약·분쟁이 있으면 종료 시점까지 연장하고 사유를 기록한다.
- 보유기간 계산에 필요한 퇴직일, 계약 종료일, 마지막 투입 종료일과 파기 예정일을 구조화하여 관리한다.

## 5. 파기 운영

- 매월 파기 대상 후보를 산출하고 업무 소유자와 개인정보 보호책임자의 승인 후 자동 또는 일괄 파기한다.
- 법적 분쟁·조사로 보존이 필요한 경우 `LEGAL_HOLD`와 사유·승인자·재검토일을 기록하고 해당 항목만 파기를 보류한다.
- 물리삭제가 참조 무결성을 훼손하면 대체키 치환, 값 삭제 또는 비식별화를 적용하고 재식별 연결정보를 제거한다.
- 파기 실행 결과는 대상, 근거, 건수, 실행자, 일시만 감사로그로 3년 보관하며 파기된 개인정보 값은 기록하지 않는다.
- 최소 연 1회 개인정보 카탈로그와 실제 DB·백업·로그의 보유기간을 점검한다.

## 6. 데이터 모델·시스템 경계 영향

- 직원의 `퇴직일자`, 외주인력의 `활동종료일자` 또는 외주계약 종료 참조, `개인정보파기예정일`, `LEGAL_HOLD` 여부·사유·재검토일이 있어야 보유기간을 자동 계산할 수 있다.
- 현재 인력 물리 모델에는 직원 재직상태만 있고 퇴직일자가 없으며 외주인력 종료일도 없다. 인력 기능 상세설계에서 컬럼 또는 별도 개인정보 생명주기 엔터티를 추가한다.
- v1.0에 외주계약 엔터티가 없으면 마지막 투입 종료일과 프로젝트 최종 종료일 중 늦은 날을 임시 종료기준일로 사용하고, 외주계약 도입 시 계약 종료일을 계산식에 추가한다.
- 근로·세무·외주계약 증빙의 원본 시스템이 별도 HR·회계·문서 시스템이면 BMS에 증빙을 중복 저장하지 않는다. BMS는 원본 시스템과 보존 근거만 참조하고 운영용 프로필의 파기 기준을 독립 적용한다.

## 7. 법적 기준 메모

- 개인정보 보호법 제15조: 동의, 법령상 의무, 계약 이행 또는 정보주체의 권리보다 명백히 우선하는 정당한 이익 등 수집·이용 근거.
- 개인정보 보호법 제21조: 불필요해진 개인정보의 지체 없는 파기와 법정 보존분의 분리 저장·관리.
- 근로기준법 제42조 및 시행령 제22조: 근로자 명부와 근로계약 관련 중요 서류의 3년 보존.
- 국세기본법 제85조의3: 장부와 증거서류의 원칙적 5년 보존.

법령 개정과 개별 사실관계에 따라 달라질 수 있으므로 운영 전 개인정보 보호책임자 또는 법무·노무·세무 담당자의 확인을 받는다.
