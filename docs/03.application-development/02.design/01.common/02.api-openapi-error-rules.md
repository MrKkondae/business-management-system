# API·OpenAPI·오류 공통 규칙

## 1. 기본 계약

| 항목 | 기준 |
| --- | --- |
| 기본 경로 | `/api/v1` |
| 미디어 타입 | `application/json`; 파일 API만 별도 타입 허용 |
| URL | 소문자 복수 명사와 kebab-case, 동사형 URL은 상태 명령에만 제한 |
| Method | 조회 `GET`, 생성 `POST`, 부분 변경 `PATCH`, 전체 교체 `PUT`, 삭제 `DELETE` |
| 날짜 | `YYYY-MM-DD` |
| 날짜·시간 | ISO 8601 UTC(`Z`)로 송수신하고 화면에서 Asia/Seoul로 표시 |
| 식별자 | 문자열로 계약하여 JavaScript 정밀도 손실을 방지 |
| 성공 응답 | 공통 래퍼 없이 자원 DTO 또는 처리 결과 DTO를 직접 반환 |
| 빈 성공 응답 | 응답 본문이 불필요하면 `204 No Content` |

생성 성공은 `201 Created`와 `Location` 헤더를 사용한다. 요청 문법·필드 검증은 `400`, 인증 없음·만료는 `401`, 권한 없음은 `403`, 자원 없음은 `404`, 중복·상태·동시성 충돌은 `409`, 서버 오류는 `500`을 기본으로 한다.

## 2. 목록·검색

| 항목 | 기준 |
| --- | --- |
| 페이지 | 0부터 시작하는 `page`, 기본 0 |
| 크기 | `size`, 기본 20, 최대 100 |
| 정렬 | `sort=field,asc` 반복 허용; API별 허용 필드 목록 적용 |
| 검색 문자열 | 앞뒤 공백 제거, 빈 문자열은 조건 없음으로 처리 |
| 기간 검색 | 시작일 포함, 종료일 포함; 날짜·시간 범위는 `[from, to)` 권장 |

목록 응답은 아래 구조를 사용한다.

```json
{
  "items": [],
  "page": 0,
  "size": 20,
  "totalElements": 0,
  "totalPages": 0,
  "sort": ["createdAt,desc"]
}
```

클라이언트가 보낸 임의 컬럼명을 정렬식이나 검색식에 직접 연결하지 않는다. 대량 다운로드는 목록 API와 분리하고 별도 권한과 감사 로그를 적용한다.

## 3. 오류 계약

```json
{
  "code": "COMMON_VALIDATION_FAILED",
  "message": "입력값을 확인해 주세요.",
  "traceId": "01J...",
  "fieldErrors": [
    {"field": "projectName", "reason": "REQUIRED", "message": "프로젝트명은 필수입니다."}
  ]
}
```

- 오류코드는 `DOMAIN_REASON` 형태의 대문자 snake case로 만들고 의미를 변경해 재사용하지 않는다.
- `message`는 사용자에게 표시할 안전한 한국어 문장이다. 분기 처리는 메시지가 아니라 `code`로 한다.
- 필드 검증 오류가 아니면 `fieldErrors`는 빈 배열 또는 생략으로 통일하며 OpenAPI에서 명시한다.
- 내부 예외명, 스택 추적, SQL, 테이블명, 개인정보와 인증정보는 응답하지 않는다.
- 예상 가능한 업무 거절은 `WARN`, 입력 오류와 정상적인 401·403·404는 `INFO` 또는 `WARN`, 서버 장애는 `ERROR`로 기록한다. 동일 오류를 여러 계층에서 중복 기록하지 않는다.

| 공통 코드 | HTTP | 의미 |
| --- | :---: | --- |
| `COMMON_VALIDATION_FAILED` | 400 | 요청 필드 검증 실패 |
| `AUTH_AUTHENTICATION_REQUIRED` | 401 | 인증 없음 또는 세션 만료 |
| `AUTH_LOGIN_FAILED` | 401 | ID 존재 여부를 숨긴 로그인 실패 |
| `AUTH_FORBIDDEN` | 403 | 기능 또는 데이터 접근권한 없음 |
| `AUTH_CSRF_INVALID` | 403 | CSRF 토큰 없음 또는 불일치 |
| `AUTH_REAUTHENTICATION_REQUIRED` | 403 | 중요 작업을 위한 최근 비밀번호 재인증 필요 |
| `AUTH_PASSWORD_CHANGE_REQUIRED` | 403 | 초기 비밀번호 제한 세션에서 허용되지 않은 API 접근 |
| `AUTH_TOO_MANY_ATTEMPTS` | 429 | 로그인 요청 속도 제한 초과 |
| `COMMON_RESOURCE_NOT_FOUND` | 404 | 자원 없음 |
| `COMMON_DUPLICATE_RESOURCE` | 409 | 유일성 충돌 |
| `COMMON_INVALID_STATE` | 409 | 허용되지 않은 상태 전이 |
| `COMMON_CONCURRENT_MODIFICATION` | 409 | 낙관적 잠금 충돌 |
| `COMMON_INTERNAL_ERROR` | 500 | 노출 가능한 상세가 없는 서버 오류 |

업무 오류는 기능 설계와 오류 카탈로그에 추가하고 OpenAPI 응답에 연결한다.

## 4. OpenAPI 원본 규칙

- `../04.openapi/openapi.yaml`을 API 계약의 단일 원본으로 사용한다.
- `operationId`는 `{domain}{Action}{Resource}` lowerCamelCase로 유일하게 작성한다.
- 공통 `PageMetadata`, `Problem`, `FieldError`, 세션 쿠키와 CSRF 헤더를 `components`에 정의한다.
- 보호 API는 세션 보안 요구사항을 명시하고 상태 변경 API에는 CSRF 헤더를 함께 요구한다.
- 모든 작업은 요구사항 ID, BFD ID, 화면 ID, 권한을 `x-bms-*` 확장 속성으로 추적한다.
- Controller와 TypeScript Client를 계약보다 먼저 임의 작성하지 않는다. CI에서 OpenAPI 문법과 `operationId` 중복을 검증한다.
- 호환되지 않는 필드 삭제·타입 변경·필수 전환은 `/api/v2` 또는 승인된 전환 계획 없이는 허용하지 않는다.
