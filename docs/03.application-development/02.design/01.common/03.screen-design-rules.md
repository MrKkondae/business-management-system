# 화면 공통 규칙

## 1. 목록 화면

- 최초 페이지는 0, 화면 표시 페이지는 1부터 보여주고 API 호출 시 변환한다.
- 기본 20건, 선택 크기는 20·50·100건으로 제한한다.
- 조회조건과 페이지·정렬은 URL query parameter로 보존하여 새로고침과 뒤로가기를 지원한다.
- 검색 실행 시 첫 페이지로 이동한다. 초기화는 조건과 정렬을 화면 기본값으로 되돌린다.
- 로딩 중 기존 목록을 임의로 비우지 않고 진행 상태를 표시한다.
- 0건은 정상적인 빈 상태와 다음 행동을 안내하며 오류 화면으로 처리하지 않는다.
- 개인정보 컬럼은 개인정보 처리 기준에 따라 마스킹하고 전체보기·다운로드 권한을 분리한다.

## 2. 상세·폼

- 등록과 수정은 같은 검증 규칙을 공유하되 수정 불가 항목을 명시한다.
- 필수 항목은 레이블과 접근성 속성으로 표시하고 색상만으로 구분하지 않는다.
- 형식 검증은 입력 종료 또는 제출 시 제공하고, 업무 규칙 오류는 API의 오류코드로 표시한다.
- 저장 중 버튼을 비활성화하여 중복 제출을 막고 완료 후 상세 화면으로 이동하거나 성공 메시지와 함께 재조회한다.
- 변경사항이 있는 상태에서 이탈하면 확인한다.
- 삭제·비활성·완료·취소처럼 영향이 큰 명령은 대상명과 영향을 포함한 확인창을 사용한다.
- 서버가 반환한 권한과 현재 상태를 기준으로 버튼을 숨기거나 비활성화하되, 서버 검사를 대체하지 않는다.

## 3. 공통 상태

| 상태 | 처리 |
| --- | --- |
| 로딩 | 스켈레톤 또는 진행 표시, 중복 명령 차단 |
| 빈 결과 | “조회 결과가 없습니다”와 조건 초기화·등록 등 가능한 행동 제공 |
| 검증 실패 | 필드 인접 메시지와 첫 오류 포커스 이동 |
| 인증 만료 | 로그인 화면 이동, 저장되지 않은 값의 복원은 보장하지 않음을 안내 |
| 권한 없음 | 403 전용 안내, 메뉴로 복귀 제공 |
| 찾을 수 없음 | 404 전용 안내 |
| 서버 오류 | `traceId`와 재시도 방법 표시, 내부 상세는 숨김 |
| 동시 수정 | 최신 데이터 재조회 후 다시 변경하도록 안내 |

날짜는 `YYYY-MM-DD`, 화면 날짜·시간은 `YYYY-MM-DD HH:mm` Asia/Seoul로 표시한다. 모든 입력 요소는 키보드 접근과 명시적 레이블을 제공한다.
