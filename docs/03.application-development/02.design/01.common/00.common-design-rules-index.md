# 공통 설계 규칙 색인

## 1. 목적

본 디렉터리는 모든 기능 설계와 구현에 적용하는 공통 계약의 원본을 관리한다. 기능별 설계가 본 규칙과 다르면 기능 문서에 사유를 쓰고 ADR로 승인받는다.

## 2. 문서 목록

| 문서 | 적용 범위 | 상태 |
| --- | --- | --- |
| `01.authentication-authorization-design.md` | 로그인, 세션, 계정, RBAC | 확정 |
| `01-1.permission-matrix.md` | 초기 역할, 메뉴·API·데이터 권한 연결 | 확정 |
| `02.api-openapi-error-rules.md` | API, OpenAPI, 공통 오류 | 확정 |
| `03.screen-design-rules.md` | 목록, 상세, 폼과 공통 화면 상태 | 확정 |
| `04.audit-log-policy.md` | 접속, 감사, 시스템, 외부연계 로그 | 확정 |
| `05.privacy-data-policy.md` | 개인정보 분류, 마스킹, 다운로드, 보유·파기 | 확정 |
| `06.database-migration-rules.md` | Flyway, 파일명, 불변성, 기준 데이터 | 확정 |
| `07.test-rules.md` | 테스트 수준, 도구, 필수 통과 기준 | 확정 |

OpenAPI 계약 원본은 `../04.openapi/openapi.yaml`에서 관리한다.

## 3. 우선순위

법령과 회사의 승인된 개인정보 처리방침을 최우선으로 하고, ADR, 본 공통 규칙, 기능별 상세설계 순으로 적용한다. 법령 또는 회사 정책이 변경되면 개인정보 보호책임자의 확인을 거쳐 본 문서와 구현을 함께 갱신한다.
