# 인증 선행 데이터와 최초 관리자 부트스트랩 설계

## 1. 목적

빈 데이터베이스에서도 인증·권한 기능을 안전하게 시작할 수 있도록 기준 데이터 반영과 최초 시스템관리자 생성을 분리한다.

- 역할, 메뉴, 역할-메뉴 관계, 공통코드는 Flyway로 반영한다.
- 최초 조직, 직원, 사용자와 사용자-역할 관계는 배포 전용 부트스트랩 명령으로 생성한다.
- 운영자가 DB에 사용자나 비밀번호를 직접 INSERT하지 않는다.

선행 데이터의 설계 원본은 `../02.catalog/seed-data/README.md`와 같은 디렉터리의 CSV 파일이다.

## 2. 실행 순서

1. Flyway가 스키마를 생성한다.
2. 검증된 초기 데이터 마이그레이션이 기본 역할, 메뉴, 권한 관계와 공통코드를 반영한다.
3. 배포 작업자가 애플리케이션의 `bootstrap-admin` 전용 실행 모드로 최초 관리자를 생성한다.
4. 정상 기동 후 최초 관리자가 임시 비밀번호로 로그인한다.
5. 제한 세션에서 비밀번호를 변경하고 등록을 완료한다.
6. 이후 사용자는 시스템의 사용자 관리 기능으로만 등록한다.

`bootstrap-admin`은 일반 애플리케이션 HTTP 포트를 열지 않으며 공개 API를 제공하지 않는다.

## 3. 선행 조건

- Flyway 마이그레이션이 모두 성공해야 한다.
- 보호 역할인 시스템관리자 역할이 정확히 한 건 존재해야 한다.
- 시스템관리자가 모든 유효 메뉴에 대한 역할-메뉴 관계를 가져야 한다.
- `COMPANY`, 계정 상태, 비활성 사유, 로그인 실패 사유, 로그아웃 유형,
  인력 구분과 재직 상태 코드가 존재해야 한다.
- 사용자 테이블이 비어 있어야 한다.

하나라도 충족하지 않으면 생성 작업을 시작하지 않고 원인을 식별할 수 있는 오류코드로 종료한다.

## 4. 입력값

| 입력 | 필수 | 처리 규칙 |
|---|---|---|
| 기본 조직명 | Y | 최상위 조직명으로 저장 |
| 사원번호 | Y | 일반 사용자 등록과 같은 중복 검증 적용 |
| 관리자 성명 | Y | 직원 성명으로 저장 |
| 로그인ID | Y | 앞뒤 공백 제거 후 소문자로 변환하여 저장 |
| 임시 비밀번호 | Y | 24시간 유효, 명령행 인자로 받지 않음 |
| 이메일·휴대전화 | N | 개인정보 정책과 같은 정규화 적용 |

최상위 조직 유형은 초기 공통코드 `COMPANY`로 고정한다. 조직명과 관리자 인적 정보는 환경별 값이므로 CSV에 넣지 않고 실행 시 입력한다.

임시 비밀번호는 배포 비밀 저장소가 프로세스 환경에 일회성으로 주입한다. 값은 표준 출력, 오류 메시지, 실행 이력, 애플리케이션 로그와 감사로그에 기록하지 않고 해시 계산 후 참조를 폐기한다.

## 5. 트랜잭션 처리

부트스트랩 애플리케이션 서비스는 다음 작업을 하나의 DB 트랜잭션으로 처리한다.

1. 시스템관리자 역할 행을 잠그고 사용자 존재 여부를 다시 확인한다.
2. 최상위 기본 조직을 생성한다.
3. 조직과 직원에 필요한 공통 업무대상·리소스 관계를 생성한다.
4. 관리자 직원정보를 생성한다.
5. 직원에 연결된 사용자 계정을 생성하고 보안버전을 `1`로 초기화한다.
6. 임시 비밀번호를 해시하여 저장하고 만료시각을 현재시각 + 24시간으로 설정한다.
7. 사용자에게 시스템관리자 역할을 부여한다.
8. 부트스트랩 성공 감사 이벤트를 기록한다.

중간 단계가 실패하면 전체를 롤백한다. 동시에 두 번 실행되더라도 역할 행 잠금 후 사용자 존재 여부를 재검사하여 한 건만 성공할 수 있다.

## 6. 최초 관리자 상태

- 계정은 활성 상태로 생성한다.
- 최초 등록 완료 여부는 미완료로 생성한다.
- 임시 비밀번호 만료 전에는 최초 등록 전용 제한 세션만 발급한다.
- 제한 세션은 유휴 10분, 절대 30분이며 비밀번호 변경과 등록 완료 API만 호출할 수 있다.
- 비밀번호 변경이 완료되면 기존 제한 세션을 폐기하고 일반 세션으로 승격한다.
- 시스템관리자 역할은 모든 유효 메뉴와 명시적 RBAC 관계를 가지며 권한 검사 우회는 사용하지 않는다.

## 7. 재실행과 복구

- 사용자가 한 건이라도 있으면 부트스트랩 재실행을 거부한다.
- 성공한 부트스트랩을 초기화하거나 삭제하는 명령은 제공하지 않는다.
- 실패하여 트랜잭션이 롤백된 경우 입력과 선행 조건을 수정한 뒤 다시 실행할 수 있다.
- 운영 데이터가 일부 수동 입력된 상태에서는 자동 보정하지 않고 운영 복구 절차로 전환한다.

## 8. 검증 항목

| 검증 | 기대 결과 |
|---|---|
| 초기 데이터 CSV 검증 | 참조, 중복, 코드 길이, 시스템관리자 전체 메뉴 권한 오류 0건 |
| 빈 DB 마이그레이션 | 역할·메뉴·권한·공통코드가 한 번씩 생성됨 |
| 최초 실행 | 조직·직원·사용자·역할 관계가 모두 생성됨 |
| 동시 실행 | 한 실행만 성공하고 중복 데이터가 없음 |
| 실패 주입 | 모든 업무 데이터가 롤백됨 |
| 재실행 | 사용자 존재 오류로 거부됨 |
| 비밀값 검사 | 로그·SQL·CSV·프로세스 인자에 비밀번호가 없음 |
| 최초 로그인 | 제한 세션만 발급되고 24시간 후 임시 비밀번호가 거부됨 |

## 9. 구현 전제

- CSV 검증 명령은 `python scripts/validate_seed_data.py`이다.
- SQL 생성 명령은 `python scripts/generate_seed_migration.py --output <신규 마이그레이션 경로>`이다.
- CSV에서 생성한 SQL은 반드시 신규 Flyway 버전 파일로 검토·반영한다.
- 부트스트랩 실행 명령, 비밀값 변수명과 운영 배포 예시는 구현 단계의 배포 런북에서 확정한다.
