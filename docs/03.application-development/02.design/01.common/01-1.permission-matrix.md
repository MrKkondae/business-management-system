# 권한 매트릭스 공통 기준

## 1. 초기 역할과 메뉴 권한

초기 역할, 운영 메뉴와 역할-메뉴 관계의 설계 원본은 `../02.catalog/seed-data/`의 CSV 파일이다.

- 시스템관리자는 운영 DB에 등록된 모든 유효 메뉴에 대한 명시적 권한 관계를 가진다.
- 새 메뉴를 추가할 때 시스템관리자 권한 관계도 같은 마이그레이션에 추가한다.
- 시스템관리자도 일반 RBAC 판정 절차를 사용하며 역할명이나 계정으로 권한 검사를 우회하지 않는다.
- 다른 초기 역할의 메뉴 권한은 해당 기능의 상세설계에서 조회·변경·데이터 범위를 승인한 후 CSV와 신규 마이그레이션에 추가한다.
- 분석 단계 메뉴 권한표는 요구사항 입력이며, 실제 배포 권한의 원본은 초기 데이터 CSV와 후속 마이그레이션이다.

## 2. 판정 순서

1. 유효한 세션과 활성 계정인지 확인한다.
2. API에 연결된 화면·메뉴 중 하나를 사용자의 역할 합집합이 허용하는지 확인한다.
3. 기능이 데이터 범위 정책을 정의했다면 조직, 담당자, 프로젝트 PM 또는 본인 여부를 추가 확인한다.
4. 개인정보 원문 조회·다운로드, 권한 변경과 로그 조회는 별도 고위험 권한을 추가 확인한다.

프론트엔드의 메뉴 숨김과 버튼 제어는 편의 기능이며, 모든 API가 같은 판정을 서버에서 다시 수행한다. 자원 ID만 바꾸어 다른 사용자의 행에 접근할 수 없도록 대상 엔터티를 조회한 뒤 데이터 권한을 검사한다.

시스템관리자 역할 부여·회수, 역할 메뉴권한 변경·삭제, 다른 사용자 비밀번호 초기화와 관리자의 계정 활성·비활성 등 중요 작업은 로그인 또는 현재 비밀번호 재인증 후 10분 안에서만 허용한다. 역할·메뉴권한 변경은 대상 사용자의 보안버전을 증가시키고 기존 세션을 다음 보호 요청에서 종료한다.

## 3. 추적 규칙

각 기능 API는 `function_id`, `screen_id`, `menu_id`, 허용 역할, 데이터 범위 정책과 개인정보 권한을 명시한다. 공통 호출 API는 호출 가능한 화면 목록 또는 전용 공통 권한을 가진다. 권한 매트릭스 변경은 역할·메뉴 기준 데이터 마이그레이션, OpenAPI 확장 속성, 테스트와 함께 변경한다.
