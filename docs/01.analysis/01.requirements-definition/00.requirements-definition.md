# 요구사항 정의서

## 1. 문서 개요

### 1.1 목적

본 문서는 **BMS(Business Management System)** 구축을 위한 업무 요구사항, 기능 요구사항 및 비기능 요구사항을 정의하는 것을 목적으로 한다.

본 문서는 향후 업무 분석, 화면 설계, 데이터 모델(ERD), 시스템 설계의 기준 문서로 활용한다.

### 1.2 적용 범위

본 시스템은 소규모 SI/SM 업체의 영업, 제안, 계약, 프로젝트 수행, 인력 운영, 예산 및 원가관리, 매출 및 청구 업무를 지원하는 내부 업무 시스템을 대상으로 한다.

---

# 2. 시스템 개요

## 2.1 시스템명

Business Management System (BMS)

## 2.2 구축 목적

회사 내 고객, 영업기회, 제안, 계약, 프로젝트 수행 현황을 통합 관리하고, 프로젝트 예산 및 원가를 체계적으로 관리하여 사업별 손익을 분석함으로써 경영 의사결정을 지원한다.

또한 프로젝트 수행 과정에서 발생하는 인력 투입, 원가, 매출, 청구 정보를 통합 관리하여 업무 효율성과 데이터 신뢰성을 향상시키는 것을 목표로 한다.

## 2.3 주요 사용자

| 사용자      | 설명                            |
| -------- | ----------------------------- |
| 대표/임원    | 전체 영업, 프로젝트, 원가, 손익, 매출 현황 조회 |
| 영업 담당자   | 고객사, 영업기회, 제안 진행 관리           |
| PM/PL    | 프로젝트, 일정, 인력 투입, 원가, 이슈 관리    |
| 경영지원 담당자 | 계약, 예산, 원가, 매출, 청구, 수금 관리     |
| 일반 직원    | 본인 프로젝트, 일정, 업무 현황 조회         |
