# 제품 백로그 (Product Backlog)

## 1. 문서 개요

본 문서는 프로젝트 관리 시스템의 기능 요구사항 및 개선사항을 관리하기 위한 제품 백로그이다.

백로그는 프로젝트 진행 중 지속적으로 추가, 변경 및 우선순위 조정이 가능하며, 개발 완료 시까지 지속적으로 관리한다.

---

## 2. 개발 로드맵

### v1.0 (MVP)

**목표**

회사에서 실제 사용할 수 있는 최소 기능을 제공한다.

#### 주요 기능

* 로그인 및 사용자 관리
* 고객사 관리
* 영업기회 관리
* 프로젝트 관리
* 인력 관리
* 기본 대시보드

---

### v1.1

**목표**

프로젝트 운영 기능을 강화한다.

#### 주요 기능

* 계약 관리
* 매출 관리
* 청구 및 수금 관리
* 알림 기능
* 권한 관리 고도화

---

### v1.2

**목표**

사용 편의성을 개선한다.

#### 주요 기능

* 메모 기능
* 첨부파일 관리
* 통합 검색
* 엑셀 다운로드
* 프로젝트 통계

---

### v2.0

**목표**

AI 기반 업무 지원 기능을 제공한다.

#### 주요 기능

* AI 문서 생성
* AI 회의록 요약
* AI 일정 추천
* AI 프로젝트 위험도 분석
* AI 보고서 자동 생성

# 4. 상태 정의

| 상태   | 설명               |
| ---- | ---------------- |
| 미진행  | 개발을 시작하지 않은 상태   |
| 분석중  | 요구사항 분석 중        |
| 설계중  | 설계 진행 중          |
| 개발중  | 개발 진행 중          |
| 테스트중 | 테스트 진행 중         |
| 완료   | 개발 및 테스트 완료      |
| 보류   | 우선순위 변경 또는 일정 보류 |

---

# 5. 우선순위 정의

| 우선순위   | 설명               |
| ------ | ---------------- |
| High   | MVP 개발 대상(필수 기능) |
| Medium | 기능 확장 대상         |
| Low    | 장기 개선 및 신규 기능    |
