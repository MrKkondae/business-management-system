# ADR-001 공통 인터페이스 기준선

## 상태

승인

## 일자

2026-07-21

## 배경

첫 번째 수직 기능 구현 전에 API 응답, 시간대, 인증·권한, 개인정보, 로그, 데이터베이스 변경과 테스트 기준을 고정해야 기능별 계약이 서로 달라지는 것을 방지할 수 있다.

## 결정

- 성공 API는 공통 래퍼 없이 DTO를 직접 반환하고 오류만 `code`, `message`, `traceId`, `fieldErrors` 공통 구조를 사용한다.
- 목록은 0 기반 페이지, 기본 20건, 최대 100건을 사용한다.
- OpenAPI 3.1 명세를 Controller와 TypeScript Client의 계약 원본으로 사용한다.
- 서버 저장과 API 날짜·시간은 UTC, 화면 표시는 Asia/Seoul을 사용한다.
- 인증은 서버 세션과 CSRF, 권한은 역할–메뉴 기반 RBAC를 적용한다.
- 비밀번호는 12~64자, 취약·재사용 금지, 초기 1회용 변경을 적용하고 일반적인 정기 강제 변경은 하지 않는다.
- PostgreSQL 변경은 Flyway로만 수행하고 JPA는 `ddl-auto=validate`를 사용한다.
- 인력 개인정보는 적법한 처리 근거와 목적이 있는 경우 운영 목적 종료 후 최대 1년을 기본안으로 하고, 근로 증빙 3년, 세무·계약 증빙 5년처럼 목적과 법적 근거별로 분리 보관한다. 근거가 없거나 목적이 먼저 끝나면 지체 없이 파기한다.
- 공통 테스트 수준과 필수 보안·개인정보·업무규칙 시나리오를 품질 게이트로 사용한다.

## 영향

- 기능별 설계는 공통 규칙을 반복하지 않고 해당 규칙과 OpenAPI 공통 schema를 참조한다.
- 공통 기준의 예외는 기능 문서에 사유를 적고 별도 ADR 승인을 받아야 한다.
- 실제 개인정보 보유기간은 운영 전 회사 개인정보 처리방침과 법무·노무·세무 검토를 거쳐 확정한다.
