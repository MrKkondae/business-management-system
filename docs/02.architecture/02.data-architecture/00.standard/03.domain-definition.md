# BMS 도메인 정의서

## 1. 문서 개요

### 1.1 목적

본 문서는 BMS(Business Management System) 데이터베이스 설계 시 사용하는 표준 도메인(Domain)을 정의한다.

도메인은 데이터의 형식(Type), 길이(Length), 정밀도(Precision), Null 허용 여부 등을 표준화하여 데이터의 일관성과 재사용성을 확보하기 위한 기준으로 활용한다.

본 문서는 테이블 정의서, ERD, 표준용어 사전의 기준 문서로 사용한다.

---

## 1.2 작성 원칙

* 동일한 의미의 데이터는 동일한 도메인을 사용한다.
* 컬럼은 가능한 표준 도메인을 사용한다.
* 신규 도메인은 기존 도메인 재사용 여부를 우선 검토한다.
* DBMS 변경 시에도 동일한 도메인명을 사용한다.

# 2. 문자형 도메인

| 도메인명   | PostgreSQL Type | Null | Default | 설명                                          | 예시      |
| ---------- | --------------- | ---- | ------- | --------------------------------------------- | --------- |
| ID         | VARCHAR(30)     | N    |         | 식별자                                        | PRJ_ID    |
| CODE       | VARCHAR(20)     | N    |         | 공통코드                                      | STATUS_CD |
| NAME       | VARCHAR(100)    | N    |         | 일반 명칭(직원명, 대표자명, 코드명)           | EMP_NM    |
| LONG_NAME  | VARCHAR(200)    | N    |         | 일반 명칭(프로젝트명, 계약명, 고객사명)       | PRJ_NM    |
| TITLE      | VARCHAR(200)    | N    |         | 공지사항 제목, 게시글 제목                    |           |
| SHORT_DESC | VARCHAR(500)    | Y    |         | 짧은 설명                                     | REMARK    |
| CONTENT    | TEXT            | Y    |         | 긴 내용                                       | CONTENT   |
| NUMBER     | VARCHAR(30)     | Y    |         | 업무번호(계약번호, 프로젝트번호, 문서번호 등) |           |

# 11. 도메인 관리 원칙

* 신규 컬럼은 기존 도메인을 우선 사용한다.
* 동일한 데이터 형식은 동일한 도메인을 사용한다.
* 도메인 변경 시 표준용어 및 테이블 정의서를 함께 검토한다.
* DBMS별 데이터 타입은 물리 모델 단계에서 조정할 수 있으나 도메인명은 변경하지 않는다.
