
# BMS 표준단어 사전

## 1. 문서 개요

### 1.1 목적

본 문서는 BMS(Business Management System) 데이터베이스 설계 시 사용하는 표준단어(Standard Word)를 정의한다.

표준단어는 테이블명, 컬럼명, API, 프로그램, 문서 등 시스템 전반에서 동일한 용어를 사용하기 위한 기준으로 활용한다.

---

## 1.2 작성 원칙

- 동일한 의미는 하나의 표준단어만 사용한다.
- 영문명은 Full Name을 사용한다.
- 컬럼명 작성 시에는 약어(Abbreviation)를 사용한다.
- 약어는 가능한 국제적으로 많이 사용하는 약어를 사용한다.
- 의미가 중복되는 단어는 등록하지 않는다.

예)

| 한글     | 영문     | 약어 |
| -------- | -------- | ---- |
| 프로젝트 | PROJECT  | PRJ  |
| 고객     | CUSTOMER | CUST |
| 계약     | CONTRACT | CNTR |

## 6. 작성 규칙

컬럼명은 표준단어를 조합하여 작성한다.

예)

| 한글         | 컬럼명         |
| ------------ | -------------- |
| 프로젝트ID   | PRJ_ID         |
| 프로젝트명   | PRJ_NM         |
| 계약금액     | CNTR_AMT       |
| 계약상태코드 | CNTR_STATUS_CD |
| 고객ID       | CUST_ID        |
| 등록일시     | REG_DTM        |
