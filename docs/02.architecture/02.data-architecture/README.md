# BMS 데이터 아키텍처

이 디렉토리는 BMS의 데이터 표준, 논리 모델, 물리 모델, 모델 다이어그램 및 주요 설계 결정을 관리한다.

## 디렉토리 구성

- 디렉토리: `00.governance`; 역할: 데이터 아키텍처 정책, 명명규칙, 관리 및 검토 절차
- 디렉토리: `01.standard`; 역할: 표준단어, 표준용어, 논리 도메인 및 DBMS 타입 매핑
- 디렉토리: `02.logical-model`; 역할: 엔터티, 속성, 관계 및 논리 모델 설명
- 디렉토리: `03.physical-model`; 역할: 테이블, 컬럼, 제약조건, 인덱스 및 DBMS 예외
- 디렉토리: `04.model-diagrams`; 역할: 전체 및 업무영역별 데이터 모델 다이어그램
- 디렉토리: `90.decisions`; 역할: 데이터 모델의 주요 결정과 변경 근거

## 현재 관리 상태

데이터 카탈로그의 CSV 전환이 완료되었다. 표준단어, 표준용어, 도메인, DBMS 타입 매핑, 엔터티, 엔터티 속성 및 엔터티 관계의 원본은 CSV이다. Markdown은 규칙, 원칙, 설명과 CSV 위치 안내만 관리한다.

## 기본 관리 원칙

- 규칙과 설명은 Markdown으로 관리한다.
- 목록형 데이터는 CSV에서만 관리한다.
- 논리 모델과 DBMS 종속 물리 모델을 분리한다.
- 하나의 사실은 하나의 원본에서만 관리한다.
- Excel은 원본으로 사용하지 않고 필요한 경우 원본에서 생성한다.
- 변경 이력은 문서 내부 표보다 Git 이력을 우선한다.
