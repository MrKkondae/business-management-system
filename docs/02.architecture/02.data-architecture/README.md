# BMS 데이터 아키텍처

이 디렉토리는 BMS의 데이터 표준, 논리 모델, 물리 모델, 모델 다이어그램 및 주요 설계 결정을 관리한다.

## 디렉토리 구성

| 디렉토리 | 역할 |
| --- | --- |
| `00.governance` | 데이터 아키텍처 정책, 명명규칙, 관리 및 검토 절차 |
| `01.standard` | 표준단어, 표준용어, 논리 도메인 및 DBMS 타입 매핑 |
| `02.logical-model` | 엔터티, 속성, 관계 및 논리 모델 설명 |
| `03.physical-model` | 테이블, 컬럼, 제약조건, 인덱스 및 DBMS 예외 |
| `04.model-diagrams` | 전체 및 업무영역별 데이터 모델 다이어그램 |
| `90.decisions` | 데이터 모델의 주요 결정과 변경 근거 |

## 현재 전환 상태

기존 Markdown 문서는 새 분류에 맞추어 이동했으며 내용은 변경하지 않았다. 표준 사전과 논리 모델의 CSV 전환은 별도 작업으로 수행한다. CSV 전환이 완료되기 전까지는 현재 Markdown 문서를 원본으로 취급한다.

## 기본 관리 원칙

- 규칙과 설명은 Markdown으로 관리한다.
- 목록형 데이터는 CSV로 전환하여 관리한다.
- 논리 모델과 DBMS 종속 물리 모델을 분리한다.
- 하나의 사실은 하나의 원본에서만 관리한다.
- Excel은 원본으로 사용하지 않고 필요한 경우 원본에서 생성한다.
- 변경 이력은 문서 내부 표보다 Git 이력을 우선한다.
