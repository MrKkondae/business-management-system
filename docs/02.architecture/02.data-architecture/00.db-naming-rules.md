# BMS DB 명명규칙

## 1. 문서 개요

### 1.1 목적

본 문서는 **BMS(Business Management System)** 데이터베이스 설계 시 적용할 명명규칙(Naming Rule)을 정의한다.

명명규칙은 데이터 모델의 일관성을 확보하고 개발 및 유지보수의 효율성을 높이기 위한 기준으로 활용한다.

본 문서는 다음 DB 객체에 대한 명명규칙을 정의한다.

* 테이블(Table)
* 컬럼(Column)
* Primary Key
* Foreign Key
* Index
* Sequence
* View
* Trigger

---

## 2. 기본 원칙

### 2.1 공통 원칙

* 모든 DB 객체명은 **영문 대문자**를 사용한다.
* 단어 구분은 **언더스코어(_)를 사용**한다.
* 공백 및 특수문자는 사용하지 않는다.
* 약어는 표준 약어를 사용한다.
* 동일한 의미의 데이터는 동일한 명칭을 사용한다.
* Oracle, MariaDB 등 DBMS에 관계없이 동일한 규칙을 적용한다.

예)

| 구분  | 사용                    |
| --- | --------------------- |
| 테이블 | `TB_PRJ_MST`          |
| 컬럼  | `PRJ_ID`              |
| PK  | `PK_TB_PRJ_MST`       |
| FK  | `FK_TB_PRJ_MEMBER_01` |

# 3. 업무영역 Prefix

업무영역을 구분하기 위해 다음 Prefix를 사용한다.

| 업무영역    | Prefix |
| ------- | ------ |
| 시스템관리   | SYS    |
| 공통      | COM    |
| 고객관리    | CUST   |
| 영업관리    | SALES  |
| 계약관리    | CNTR   |
| 프로젝트관리  | PRJ    |
| SI 프로젝트 | SIPRJ  |
| SM 프로젝트 | SMPRJ  |
| 인력관리    | RES    |
| 예산·원가관리 | COST   |
| 매출·수금관리 | REV    |

# 4. DB 객체 명명규칙

## 4.1 테이블(Table)

### 기본 형식

```text
TB_업무영역_엔터티[_유형]
```

### 예시

| 테이블명                 | 설명        |
| -------------------- | --------- |
| TB_SYS_USER          | 사용자       |
| TB_SYS_MENU          | 메뉴        |
| TB_COM_CODE          | 공통코드      |
| TB_CUST_MST          | 고객사       |
| TB_CUST_CONTACT      | 고객 담당자    |
| TB_SALES_OPPORTUNITY | 영업기회      |
| TB_CNTR_MST          | 계약        |
| TB_CNTR_CHANGE_HIST  | 계약변경이력    |
| TB_PRJ_MST           | 프로젝트      |
| TB_PRJ_MEMBER        | 프로젝트 참여인력 |
| TB_PRJ_SCHEDULE      | 프로젝트 일정   |
| TB_PRJ_ISSUE         | 프로젝트 이슈   |
| TB_PRJ_RISK          | 프로젝트 위험   |
| TB_PRJ_OUTPUT        | 프로젝트 산출물  |
| TB_RES_EMPLOYEE      | 직원        |
| TB_RES_ASSIGN        | 프로젝트 투입   |
| TB_COST_BUDGET       | 예산        |
| TB_COST_EXPENSE      | 경비        |
| TB_REV_BILL          | 청구        |
| TB_REV_RECEIPT       | 수금        |

> **업무영역 Prefix와 동일한 의미의 단어는 중복 사용하지 않는다.**
>
> 예)
>
> * ○ `TB_PRJ_MST`
> * × `TB_PRJ_PROJECT_MST`

# 5. 컬럼 명명규칙

컬럼명은 **업무대상 + 속성** 형태를 사용한다.

## 예시

| 한글     | 컬럼명      |
| ------ | -------- |
| 프로젝트ID | PRJ_ID   |
| 프로젝트명  | PRJ_NM   |
| 고객ID   | CUST_ID  |
| 계약ID   | CNTR_ID  |
| 직원ID   | EMP_ID   |
| 시작일    | START_DT |
| 종료일    | END_DT   |

---

## 주요 Suffix

| Suffix   | 의미        |
| -------- | --------- |
| ID       | 식별자       |
| NO       | 번호        |
| NM       | 명칭        |
| CD       | 코드        |
| TYPE     | 유형        |
| STATUS   | 상태        |
| DT       | 날짜        |
| DTM      | 일시        |
| YY       | 년도        |
| YM       | 년월        |
| AMT      | 금액        |
| UNIT_AMT | 단가        |
| QTY      | 수량        |
| CNT      | 건수        |
| RATE     | 비율        |
| MM       | Man Month |
| YN       | 여부        |
| DESC     | 설명        |
| REMARK   | 비고        |

# 6. 공통 컬럼

모든 업무 테이블은 다음 공통 컬럼을 사용한다.

| 컬럼명     | 설명   |
| ------- | ---- |
| REG_ID  | 등록자  |
| REG_DTM | 등록일시 |
| MOD_ID  | 수정자  |
| MOD_DTM | 수정일시 |
| DEL_YN  | 삭제여부 |

### 삭제 정책

* 물리 삭제를 지양한다.
* 논리 삭제를 기본으로 한다.
* 삭제 여부는 `DEL_YN` 컬럼을 사용한다.
* 값은 `Y`, `N`만 사용한다.

---

# 7. Primary Key

PK 컬럼은 엔터티 약어 + ID를 사용한다.

| 테이블             | PK      |
| --------------- | ------- |
| TB_PRJ_MST      | PRJ_ID  |
| TB_CUST_MST     | CUST_ID |
| TB_CNTR_MST     | CNTR_ID |
| TB_RES_EMPLOYEE | EMP_ID  |

### PK 제약조건

```text
PK_테이블명
```

예)

```text
PK_TB_PRJ_MST
PK_TB_CUST_MST
```

---

# 8. Foreign Key

FK 컬럼은 참조 테이블의 PK 컬럼명을 그대로 사용한다.

예)

| 참조 테이블 | FK 컬럼   |
| ------ | ------- |
| 고객     | CUST_ID |
| 프로젝트   | PRJ_ID  |
| 계약     | CNTR_ID |
| 직원     | EMP_ID  |

### FK 제약조건

```text
FK_테이블명_순번
```

예)

```text
FK_TB_PRJ_MEMBER_01
FK_TB_PRJ_MEMBER_02
```

---
