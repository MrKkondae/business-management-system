# BMS 표준용어 사전

## 1. 문서 개요

### 1.1 목적

본 문서는 BMS(Business Management System)에서 사용하는 표준용어(Standard Term)를 정의한다.

표준용어는 표준단어를 조합하여 생성하며, 데이터베이스 컬럼명, API, DTO, Entity 등 시스템 전반에서 동일한 명칭을 사용하기 위한 기준으로 활용한다.

---

## 1.2 작성 원칙

- 표준단어를 조합하여 작성한다.
- 동일한 의미는 하나의 표준용어만 사용한다.
- 컬럼명은 표준 약어를 사용한다.
- 컬럼명은 DB 명명규칙을 따른다.
- 데이터형은 도메인 정의서를 참조한다.

# 9. 작성 규칙

## 표준단어 조합

- 표준단어 1: 프로젝트; 표준단어 2: ID; 표준용어: 프로젝트ID; 컬럼명: PRJ_ID
- 표준단어 1: 프로젝트; 표준단어 2: 명; 표준용어: 프로젝트명; 컬럼명: PRJ_NM
- 표준단어 1: 프로젝트; 표준단어 2: 상태; 표준용어: 프로젝트상태; 컬럼명: PRJ_STATUS
- 표준단어 1: 프로젝트; 표준단어 2: 상태; 표준용어: 프로젝트상태코드; 컬럼명: PRJ_STATUS_CD
- 표준단어 1: 계약; 표준단어 2: 금액; 표준용어: 계약금액; 컬럼명: CNTR_AMT
- 표준단어 1: 고객; 표준단어 2: 명; 표준용어: 고객명; 컬럼명: CUST_NM

---

# 10. 관리 원칙

- 동일한 의미의 컬럼은 반드시 동일한 표준용어를 사용한다.
- 동일한 표준용어는 하나만 등록한다.
- 신규 컬럼 생성 시 표준용어 사전에 먼저 등록한 후 사용한다.
- 데이터형은 도메인 정의서를 따른다.
