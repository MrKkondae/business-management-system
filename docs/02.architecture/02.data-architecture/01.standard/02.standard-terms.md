# BMS 표준용어 사전

## 1. 문서 개요

### 1.1 목적

본 문서는 BMS(Business Management System)에서 사용하는 표준용어(Standard Term)를 정의한다.

표준용어는 표준단어를 조합하여 생성하며, 데이터베이스 컬럼명, API, DTO, Entity 등 시스템 전반에서 동일한 명칭을 사용하기 위한 기준으로 활용한다.

---

## 1.2 작성 원칙

- 표준단어를 조합하여 작성한다.
- 동일한 의미는 하나의 표준용어만 사용한다.
- 컬럼명은 표준 약어를 사용한다.
- 컬럼명은 DB 명명규칙을 따른다.
- 데이터형은 도메인 정의서를 참조한다.

---

# 2. 표준용어 원본

표준용어 데이터는 `standard-terms.csv`에서 통합 관리한다. 업무영역별 파일을 별도로 만들거나 다른 영역에서 사용하는 용어를 복제하지 않는다.

`owner_area`는 용어의 사용 범위가 아니라 용어의 의미와 변경을 책임지는 소유 업무영역이다. 예를 들어 프로젝트가 `고객ID`를 사용하더라도 이 용어의 소유 영역은 고객관리이며 한 건만 등록한다.

허용하는 소유 업무영역은 다음과 같다.

- `system`: 시스템관리
- `common`: 공통
- `customer`: 고객관리
- `sales`: 영업관리
- `contract`: 계약관리
- `project`: 프로젝트관리
- `employee`: 인력관리
- `cost`: 예산·원가관리
- `revenue`: 매출·수금관리

---

# 3. 작성 규칙

## 표준단어 조합

- 표준단어 1: 프로젝트; 표준단어 2: ID; 표준용어: 프로젝트ID; 컬럼명: PRJ_ID
- 표준단어 1: 프로젝트; 표준단어 2: 명; 표준용어: 프로젝트명; 컬럼명: PRJ_NM
- 표준단어 1: 프로젝트; 표준단어 2: 상태; 표준용어: 프로젝트상태; 컬럼명: PRJ_STATUS
- 표준단어 1: 프로젝트; 표준단어 2: 상태; 표준용어: 프로젝트상태코드; 컬럼명: PRJ_STATUS_CD
- 표준단어 1: 계약; 표준단어 2: 금액; 표준용어: 계약금액; 컬럼명: CNTR_AMT
- 표준단어 1: 고객; 표준단어 2: 명; 표준용어: 고객명; 컬럼명: CUST_NM

---

# 4. 관리 원칙

- 동일한 의미의 컬럼은 반드시 동일한 표준용어를 사용한다.
- 동일한 표준용어는 하나만 등록한다.
- 여러 업무영역에서 사용하는 용어는 복제하지 않고 소유 업무영역의 용어를 참조한다.
- 신규 용어는 사용하는 업무영역이 아니라 의미를 책임지는 업무영역을 `owner_area`로 지정한다.
- 같은 컬럼명을 사용하더라도 한글명과 업무 의미가 다르면 별도 용어로 관리할 수 있다.
- 신규 컬럼 생성 시 표준용어 사전에 먼저 등록한 후 사용한다.
- 데이터형은 도메인 정의서를 따른다.

## 업무영역별 보완 순서

표준용어는 `system`, `common`, `customer`, `sales`, `contract`, `project`, `employee`, `cost`, `revenue` 순서로 한 영역씩 검토한다.

각 영역은 엔터티 속성과 기존 표준단어·도메인을 대조하고, 미등록 용어 등록과 중복·소유 영역 검토를 마친 후 다음 영역으로 진행한다.

검증은 항상 전체 카탈로그를 대상으로 수행하되 검토용 작업 보고서는 `--review-area`로 한 영역씩 생성한다.

```powershell
python scripts/validate_data_catalog.py --review-area system --report tmp/data-catalog-validation-system.csv
```
