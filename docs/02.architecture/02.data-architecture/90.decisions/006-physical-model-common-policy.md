# ADR-006 전체 물리모델 공통 기준

## 상태

승인

## 일자

2026-07-20

## 배경

기존 표준은 ID를 `VARCHAR(30)`, 일시를 `TIMESTAMP`로 매핑했지만 ID 생성방식과 시간대 원칙을 정의하지 않았다. 모든 논리 FK를 DB에 생성할지, 코드 초기 데이터를 어떻게 배포할지, 상태·사용여부·삭제여부를 어떻게 구분할지와 논리삭제 데이터의 고유성 정책도 물리모델 작성 전에 확정할 필요가 있다.

## 결정

- 내부 엔터티 ID는 애플리케이션이 INSERT 전에 생성한 26자리 Monotonic ULID를 사용한다.
- PostgreSQL ID 타입은 `VARCHAR(26)`을 사용하고 DB 시퀀스·트리거·DEFAULT ID 생성을 사용하지 않는다.
- 일시는 `TIMESTAMP`로 저장하되 DB 저장과 연결 세션은 UTC로 통일하고 화면에서 `Asia/Seoul`로 변환한다.
- 논리 FK는 모두 물리 제약조건 메타데이터로 관리하되 기본 집행 방식은 애플리케이션으로 한다.
- 공유 PK 하위유형, 인증·권한 관계, 공통코드 종속관계와 독립 존재할 수 없는 핵심 거래 상세에는 DB FK를 생성한다.
- DB FK는 `ON DELETE RESTRICT`, `ON UPDATE RESTRICT`를 사용하고 자동 CASCADE를 금지한다.
- 필수 코드와 시스템 설정 초기 데이터는 버전 관리 DB 마이그레이션으로 배포한다.
- 상태코드는 업무 생명주기를 표현하며 유지한다.
- 사용여부와 삭제여부가 함께 있는 엔터티에서는 사용여부를 제거한다.
- 재사용할 수 없는 업무 식별자는 전체 UNIQUE, 재사용 가능한 관리키는 `DEL_YN='N'` 부분 UNIQUE로 관리한다.

## 영향

- ID, REG_ID와 MOD_ID의 PostgreSQL 타입 매핑이 `VARCHAR(26)`으로 변경된다.
- 기존 `VARCHAR(30)` ID 데이터가 있다면 ULID 형식과 길이 검증 후 마이그레이션해야 한다.
- 현재 시스템 물리모델의 사용자역할, 역할메뉴권한과 공통코드 종속관계는 DB FK로 전환한다.
- 접속로그·시스템로그 사용자 참조와 시스템로그의 일반 감사대상 다형 참조는 애플리케이션 집행을 유지한다. 첨부파일·메모의 업무대상 참조에 관한 결정은 ADR-007에서 대체한다.
- 삭제여부와 함께 존재하는 사용여부 속성, 물리 컬럼, CHECK와 인덱스 조건을 제거한다.
