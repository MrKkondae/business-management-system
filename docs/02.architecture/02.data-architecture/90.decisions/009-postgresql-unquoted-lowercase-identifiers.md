# ADR-009 PostgreSQL 비인용 소문자 식별자 사용

## 상태

승인

## 일자

2026-07-22

## 배경

BMS 데이터 표준과 물리 모델 CSV는 DB 객체명을 영문 대문자로 표기한다. PostgreSQL은
큰따옴표가 없는 식별자를 소문자로 정규화하며, 대문자 객체명을 실제로 유지하려면 모든
DDL과 SQL에서 큰따옴표를 계속 사용해야 한다. 큰따옴표 식별자는 애플리케이션 SQL 매핑, 운영 조회와
수동 점검에서 불필요한 복잡성을 만든다.

## 결정

- 데이터 카탈로그와 설계 문서의 객체명은 기존 영문 대문자 표기를 유지한다.
- PostgreSQL DDL과 애플리케이션 SQL은 큰따옴표가 없는 소문자 식별자를 사용한다.
- 큰따옴표로 감싼 대문자 객체는 생성하지 않는다.
- DDL 생성기와 스키마 일치 검증기는 CSV 객체명을 소문자로 정규화한다.
- 예약어 충돌은 식별자 인용이 아니라 명칭 변경으로 해결한다.

## 영향

- 설계명 `TB_SYS_USER`의 PostgreSQL 실제 이름은 `tb_sys_user`가 된다.
- MyBatis Mapper SQL도 소문자 비인용 식별자를 기준으로 작성한다.
- CSV와 실제 스키마 비교 시 대소문자 정규화가 필요하다.
- 기존 DB 명명규칙의 대문자 원칙은 설계 표준 표기로 계속 유효하다.
