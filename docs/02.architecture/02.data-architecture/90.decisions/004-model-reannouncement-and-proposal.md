# ADR-004 재공고와 공고별 제안 모델

## 상태

승인

## 일자

2026-07-20

## 배경

BMS의 제안 업무에서는 고객 요청에 따른 제안서 수정작업이나 동일 공고 내 반복 제출을 수행하지 않는다. 최초 공고에 제안서를 한 번 제출하고 단독입찰이면 재공고에 보완한 제안서를 다시 한 번 제출한다. 재공고 후에도 단독입찰이면 적격심사가 진행될 수 있으나 제안서를 다시 수정하지 않는다.

기존 모델은 사업공고와 영업기회를 선택적 1:1로 연결하고 제안을 영업기회에 직접 연결하여 최초 공고와 재공고별 제안을 구분할 수 없었다. 또한 적격심사 상세정보의 나라장터 API 제공 여부와 사용자 입력 주체가 확정되지 않았다.

## 결정

- 영업기회 하나에 최초 공고와 재공고를 포함한 여러 사업공고를 연결한다.
- 최초 공고와 재공고는 별도 사업공고 행으로 저장하고 재공고의 원공고ID로 연결한다.
- 사업공고영업기회연계의 공고ID는 유일하게 유지하고 영업기회ID의 유일성은 제거한다.
- 제안은 사업공고를 필수 참조하며 공고별 최대 한 건으로 제한한다.
- 재공고 제안은 최초 제안의 수정버전이 아닌 별도 제안으로 저장한다.
- 동일 공고 내 제안 버전과 수정이력은 관리하지 않는다.
- 사업성분석은 영업기회별 최대 한 건으로 제한하며 제안에서 분석ID를 중복 참조하지 않는다.
- 수주는 제안을 필수 참조하며 제안별 최대 한 건으로 제한하고 영업기회ID를 중복 저장하지 않는다.
- 적격심사는 별도 엔터티로 만들지 않고 사업공고의 입찰진행상태로 표현한다.
- 입찰 결과의 확인 출처를 구분할 수 있도록 결과확인방법코드를 관리한다.

## 영향

- 사업공고영업기회연계의 영업기회ID에는 유일성 제약을 생성하지 않는다.
- 사업성분석.영업기회ID, 제안.공고ID와 수주.제안ID에는 유일성 제약을 생성한다.
- 영업기회는 제안에서 사업공고와 사업공고영업기회연계를 통해 조회한다.
- 수주의 영업기회는 수주에서 제안, 사업공고와 사업공고영업기회연계를 통해 조회한다.
- 적격심사 점수·의견·통보일·첨부서류의 수집 또는 입력 요구가 확정되면 별도 엔터티 추가를 재검토한다.
