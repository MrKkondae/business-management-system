# ADR-007 공통 업무대상 물리화

## 상태

승인

## 일자

2026-07-21

## 배경

첨부파일과 메모는 `(업무구분, 업무ID)` 조합으로 여러 업무 엔터티를 참조하는 논리적 다형 관계로 설계되어 있다. 이 구조는 단일 물리 FK를 생성할 수 없어 대상 존재 여부를 애플리케이션에서만 검증해야 하고, 업무구분에 따라 참조할 테이블을 동적으로 선택해야 한다. 엔터티별 파일 관계 테이블을 추가하면 참조 무결성은 강화되지만 첨부 대상이 늘어날 때마다 테이블이 증가한다.

첨부파일과 메모 외의 공통 기능으로 확장할 수 있고 추가 관계 테이블 수를 최소화할 수 있도록, 기존의 논리적 업무대상을 공통 물리 엔터티로 관리할 필요가 있다.

## 결정

- 논리적으로만 사용하던 `업무대상`을 공통 물리 엔터티로 전환한다.
- 논리 엔터티명은 `업무대상`, 물리 테이블명은 `TB_COM_TASK_TGT`, PK 컬럼명은 `TASK_TGT_ID`를 사용한다.
- 업무대상은 `TASK_TGT_TYPE_CD`로 구체적인 업무 엔터티 유형을 식별한다.
- 업무대상ID는 다른 내부 엔터티 ID와 동일하게 애플리케이션에서 생성한 26자리 Monotonic ULID를 사용한다.
- 구체적인 업무 엔터티와 업무대상은 1:1 관계로 관리한다. 구체적인 업무 엔터티는 `TASK_TGT_ID`를 필수·고유 FK로 참조한다.
- 업무대상과 구체적인 업무 엔터티는 같은 트랜잭션에서 생성하며, 생성된 업무대상ID와 업무대상유형코드는 변경하지 않는다.
- 최초 적용 대상은 계약, 프로젝트, 영업기회, 계약문서, 사업공고, 제안과 인력으로 한정한다.
- 인력근무경력, 인력학력, 인력교육, 인력상훈과 인력기술자격은 인력의 하위 데이터로만 관리하며 독립 업무대상으로 등록하지 않는다. 해당 엔터티별 첨부파일과 메모 기능도 제공하지 않는다.
- 첨부파일과 메모는 기존 `TASK_SE_CD`, `TASK_ID` 대신 `TASK_TGT_ID`를 필수 FK로 참조한다.
- 하나의 첨부파일은 하나의 업무대상에만 소속한다. 여러 업무대상에서 동일 첨부파일을 공유하지 않으며 별도 첨부파일 관계 테이블을 생성하지 않는다.
- 하나의 업무대상에는 여러 첨부파일과 여러 메모를 등록할 수 있다.
- 계약문서 PDF는 계약이 아니라 계약문서를 업무대상으로 하여 첨부한다. 계약문서는 하나의 PDF 첨부파일을 참조하며, 계약문서와 첨부파일의 업무대상ID가 일치해야 한다.
- 계약 자체도 업무대상으로 등록하여 일반 계약 관련 자료와 메모를 연결할 수 있도록 확장 가능성을 유지한다.
- 업무대상, 첨부파일, 메모 및 구체적인 업무 엔터티 간 FK는 DB 제약조건으로 생성하고 `ON DELETE RESTRICT`, `ON UPDATE RESTRICT`를 적용한다.
- 구체적인 업무 엔터티가 논리삭제되어도 업무대상과 기존 첨부파일·메모는 보존한다. 업무대상에 첨부파일이나 메모가 연결되어 있으면 업무대상을 물리삭제하지 않는다.
- 업무대상 유형과 실제 업무 엔터티의 일치 여부, 대상별 접근권한 및 계약문서와 PDF 첨부파일의 업무대상ID 일치 여부는 애플리케이션에서 검증한다.
- 시스템로그의 `TGT_TYPE_CD`, `TGT_ID`는 사용자와 역할 등 공통 업무대상에 포함되지 않는 감사대상도 기록하므로 현행 일반 다형 참조를 유지한다.

## 영향

- 공통 논리·물리 모델에 업무대상 엔터티와 `TB_COM_TASK_TGT` 테이블이 추가된다.
- 첨부파일과 메모의 `TASK_SE_CD`, `TASK_ID`가 제거되고 `TASK_TGT_ID`로 대체된다.
- 최초 적용 대상 엔터티에 필수·고유 `TASK_TGT_ID`가 추가된다.
- 첨부파일과 메모의 업무대상 참조는 애플리케이션 집행 참조에서 DB FK로 전환된다.
- 업무대상 유형코드와 구체적인 엔터티의 매핑 및 유형별 권한 판정기가 애플리케이션 설계에 추가된다.
- 계약문서 PDF 등록 시 업무대상, 첨부파일과 계약문서를 하나의 트랜잭션으로 생성하고 상호 일관성을 검증해야 한다.
- 기존 `(TASK_SE_CD, TASK_ID)` 데이터가 존재하면 업무대상 생성, 대상 엔터티 매핑, 첨부파일·메모 역매핑 후 기존 컬럼을 제거하는 데이터 이행이 필요하다.
- ADR-006의 다형 업무대상 참조를 애플리케이션 집행으로 유지한다는 결정 중 첨부파일·메모에 관한 부분은 이 결정으로 대체한다. 시스템로그의 일반 감사대상 참조에는 ADR-006의 기존 원칙을 계속 적용한다.
