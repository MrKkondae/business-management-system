# ADR-008 v1.0 최소 프로젝트 기준정보 물리화

## 상태

승인

## 일자

2026-07-21

## 배경

v1.0은 계약관리 없이도 인력투입을 사용할 수 있어야 하므로 프로젝트 기본정보를 수동으로 등록·조회·수정해야 한다. 기존 논리모델은 계약에서 프로젝트를 생성한다는 전제로 계약ID를 필수로 두었고, 인력투입의 프로젝트ID는 물리 대상이 없는 애플리케이션 참조로 남아 있었다.

프로젝트 기간은 최초 계획보다 연장될 수 있다. 연장은 여러 번 발생할 수 있지만 현재 업무에서는 드물기 때문에 모든 변경 이력을 v1.0에서 별도 엔터티로 관리할 필요성은 낮다.

## 결정

- `project` 업무영역이 최소 프로젝트 기준정보의 생성·조회·수정과 상태 변경을 소유한다.
- 프로젝트번호, 고객사, 프로젝트명, 시작일자, 최초종료일자, 현재 종료일자와 상태를 필수로 관리한다.
- 프로젝트유형과 PM은 선택값으로 관리한다.
- 계약ID는 v1.0에서 선택값으로 두고 v1.1 계약관리 도입 시 기존 수동 등록 프로젝트 연결 또는 계약 기반 생성에 사용한다.
- 상태값은 `PLANNED`, `IN_PROGRESS`, `EXTENDED`, `COMPLETED`, `CANCELED`를 사용한다.
- 등록 시 최초종료일자와 현재 종료일자를 같게 저장한다. 연장 시 최초종료일자는 유지하고 현재 종료일자만 늘리며 상태를 `EXTENDED`로 변경한다.
- 연장된 프로젝트가 완료 또는 취소되면 해당 최종 상태가 현재 상태가 된다. 연장 여부는 최초종료일자와 현재 종료일자의 차이로도 확인할 수 있다.
- v1.0에서는 연장 횟수·사유·변경 전후 일자를 저장하는 별도 이력 엔터티를 두지 않는다.
- 프로젝트와 업무대상을 같은 트랜잭션에서 생성하고 인력투입의 프로젝트 참조는 DB FK로 집행한다.

## 영향

- 프로젝트 물리 모델과 최소 기준정보 관리 화면·API가 v1.0 대상에 포함된다.
- 일정, 이슈, 위험, 산출물, 진척률, 원가와 손익은 계속 제외한다.
- 연장 이력이 필요해지면 프로젝트ID, 연장순번, 변경 전 종료일자, 변경 후 종료일자, 연장사유 및 변경 감사정보를 가진 이력 테이블을 추가한다. 프로젝트 마스터의 최초종료일자와 현재 종료일자는 그대로 유지하므로 기존 모델과 API의 큰 변경 없이 확장할 수 있다.
- 본 결정은 ADR-005의 “프로젝트는 계약관리에서 생성한다”는 프로젝트 소유권 부분을 대체하며 인력 투입 무결성에 관한 나머지 결정은 유지한다.
