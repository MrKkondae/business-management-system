# ADR-003 외부 연계 응답 필드 원형 보존

## 상태

승인

## 일자

2026-07-15

## 배경

조달청 나라장터 OpenAPI 연계 응답은 연계기관이 정의한 영문 항목명과 국문 항목명을 사용한다. BMS 표준단어·표준용어로 변환하면 연계 명세와 저장 항목의 직접 대조가 어려워지고 명세 변경 영향을 추적하기 어려워진다.

또한 OpenAPI 응답을 바로 사업공고에 반영하면 검증·매핑·반영 실패 건을 독립적으로 확인하거나 API 재호출 없이 재처리할 수 없다.

## 결정

- 변경 불가 응답 원문은 `사업공고수집원문`에 보존한다.
- 원문을 파싱한 전체 응답 항목은 `사업공고연계수신`에 선적재한다.
- 연계기관이 정의한 응답 필드는 표준단어·표준용어와 BMS 컬럼 명명규칙의 예외로 하고 영문명과 국문명을 원형 그대로 보존한다.
- 외부 필드 예외은 `interface-attribute-{business-area}.csv`의 `source_attribute_name`과 `standard_term_exception_yn=Y`로 명시한다.
- 식별자, 처리상태, 오류, 감사 속성 등 BMS 내부 필드는 기존 표준화 규칙을 적용한다.
- 원문과 연계수신을 먼저 저장한 후 검증된 값만 `사업공고`에 반영한다.

## 영향

- 조달청 연계 명세와 DB 수신 항목을 직접 대조할 수 있다.
- 사업공고 반영 실패 건을 저장된 연계수신값으로 재처리할 수 있다.
- 연계 명세가 변경되면 예외 CSV의 항목을 명시적으로 추가·변경·폐기해야 한다.
- 표준화 검증기는 `standard_term_exception_yn=Y`인 외부 필드만 표준용어 미등록 오류에서 제외해야 한다.
