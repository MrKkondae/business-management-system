# ADR-002 관계 참조 검증을 물리 모델 단계로 연기

## 상태

승인

## 일자

2026-07-13

## 배경

현재 엔터티 관계 CSV의 `relation_expression`은 논리 엔터티뿐 아니라 `상위조직`, `하위조직` 같은 관계 역할명을 포함하는 자유 형식 표현이다. 역할명을 엔터티명으로 해석하면 정상적인 자기참조 관계가 미등록 엔터티 오류로 잘못 검출된다.

현재 단계에서는 물리 테이블, PK, FK 컬럼, Null 허용 여부와 제약조건이 확정되지 않았으므로 실제 참조 무결성을 검증할 수 없다.

## 결정

- 논리 관계 CSV는 관계 후보와 역할을 설명하는 문서로 유지한다.
- 현재 검증기는 관계 CSV의 파일 구조와 관계식 형식만 검사한다.
- 관계식에 포함된 명칭의 엔터티 등록 여부는 검사하지 않는다.
- 테이블과 컬럼 설계가 끝난 뒤 물리 모델에서 참조 테이블, PK·FK 컬럼, 도메인·타입, Null 허용과 카디널리티의 일관성을 검사한다.

## 영향

- `하위조직` 같은 관계 역할명은 미등록 엔터티 오류로 처리되지 않는다.
- 논리 모델 단계에서는 관계 대상 오타를 자동으로 검출하지 않는다.
- 물리 모델 검증기를 도입할 때 참조 무결성 검사를 필수로 추가해야 한다.
