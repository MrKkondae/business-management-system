# 데이터 아키텍처 AI 작업 지침

이 디렉토리를 수정하는 AI는 먼저 `README.md`와 `00.governance/01.db-naming-rules.md`를 확인한다.

## 현재 단계

- CSV 전환 전에는 기존 Markdown 문서가 원본이다.
- 빈 디렉토리의 README는 예정된 관리 경계를 설명하며 데이터 원본이 아니다.
- 명시적인 전환 작업 없이 Markdown 내용을 CSV와 중복 작성하지 않는다.

## 변경 원칙

- 기존 식별자와 명칭을 임의로 재번호하거나 일괄 정렬하지 않는다.
- 표준단어, 표준용어, 도메인, 엔터티 간 참조 일관성을 확인한다.
- 업무영역 데이터는 해당 업무영역 파일에서만 수정한다.
- 논리 모델에 DBMS 전용 타입이나 구현 예외를 직접 추가하지 않는다.
- 생성 산출물이 도입되면 생성 파일을 직접 수정하지 않는다.
- 수정 후 중복 식별자, 미등록 참조 및 명명규칙 위반을 검증한다.
