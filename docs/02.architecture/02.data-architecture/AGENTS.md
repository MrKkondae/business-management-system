# 데이터 아키텍처 AI 작업 지침

이 디렉토리를 수정하는 AI는 먼저 `README.md`와 `00.governance/01.db-naming-rules.md`를 확인한다.

## 원본 구분

- 표준단어, 표준용어, 도메인, DBMS 타입 매핑, 엔터티, 엔터티 속성, 엔터티 관계 및 물리 모델은 CSV가 원본이다.
- Markdown은 규칙, 원칙, 설명과 CSV 파일 안내만 관리한다.
- CSV 데이터를 Markdown에 표나 목록으로 중복 작성하지 않는다.
- Excel은 CSV에서 생성하는 산출물이며 원본으로 수정하지 않는다.

## 변경 원칙

- 기존 식별자와 명칭을 임의로 재번호하거나 일괄 정렬하지 않는다.
- CSV 헤더와 열 순서는 `00.governance/02.catalog-schema.md`를 따른다.
- 표준단어, 표준용어, 도메인, 엔터티 간 참조 일관성을 확인한다.
- 업무영역별로 분리된 데이터는 해당 업무영역 파일에서만 수정한다.
- 표준용어는 `01.standard/standard-terms.csv`에서 통합 관리하고 소유 업무영역은 `owner_area`로 지정한다.
- 여러 영역에서 사용하는 표준용어를 영역별로 복제하지 않는다.
- 논리 모델에 DBMS 전용 타입이나 구현 예외를 직접 추가하지 않는다.
- 물리 컬럼 타입은 논리 도메인과 DBMS 타입 매핑에서 파생하고 예외만 `data_type_override`에 기록한다.
- DB FK를 생성하지 않는 참조도 물리 제약조건 CSV에 애플리케이션 집행 참조로 기록한다.
- 생성 산출물이 도입되면 생성 파일을 직접 수정하지 않는다.
- 수정 후 중복 식별자, 미등록 참조 및 명명규칙 위반을 검증한다.
