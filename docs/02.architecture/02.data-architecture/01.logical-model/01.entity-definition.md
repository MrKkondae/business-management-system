# BMS 엔터티 정의서

## 1. 문서 개요

### 1.1 목적

본 문서는 BMS(Business Management System)의 데이터 모델 설계를 위한 논리 엔터티를 정의한다.

엔터티 정의서는 요구사항 정의서, 업무기능분해도(BFD), 상세 업무기능분해도, 업무 프로세스를 기준으로 도출한 업무 데이터 대상을 정의하며, 향후 ERD, 테이블 정의서, API 설계 및 테스트케이스 작성의 기준 문서로 활용한다.

업무기능분해도는 업무영역과 업무기능의 계층구조를 정의하며, DB 설계의 기준 문서로 활용된다고 정의되어 있다. 또한 상세 업무기능은 L3/L4 수준에서 별도 관리하므로, 본 문서는 우선 L1/L2 업무기능 기준의 1차 엔터티 후보를 정의한다. :contentReference[oaicite:0]{index=0}

### 1.2 적용 범위

본 문서는 다음 업무영역의 엔터티를 대상으로 한다.

| ---- | -------------- |

| 04   | 영업관리       |

| 08   | 예산·원가관리 |
| 09   | 매출·수금관리 |

## 2. 엔터티 도출 원칙

### 2.1 엔터티 도출 기준

엔터티는 다음 기준에 따라 도출한다.

| 기준             | 설명                                                   |
| ---------------- | ------------------------------------------------------ |
| 업무 관리 대상   | 업무에서 등록, 수정, 삭제, 조회되는 주요 데이터        |
| 독립 식별 가능성 | 고유 식별자를 가질 수 있는 데이터                      |
| 지속 관리 여부   | 일시적인 조회 결과가 아니라 지속적으로 관리되는 데이터 |
| 업무 연계성      | 하나 이상의 업무영역 또는 업무기능에서 참조되는 데이터 |
| 이력 필요성      | 상태 변경, 변경 이력, 처리 이력 관리가 필요한 데이터   |

### 2.2 엔터티 제외 기준

다음 항목은 원칙적으로 별도 엔터티로 정의하지 않는다.

| 구분           | 설명                                      |
| -------------- | ----------------------------------------- |
| 단순 조회 결과 | 대시보드, 현황, 통계 화면의 조회 결과     |
| 집계 결과      | 원가현황, 매출현황, 손익현황 등 계산 결과 |
| 임시 데이터    | 화면 처리 과정에서만 사용하는 임시 데이터 |
| 중복 데이터    | 다른 엔터티로부터 도출 가능한 데이터      |

단, 성능, 이력 보존, 마감 처리 등의 사유가 있는 경우 집계성 엔터티를 별도로 둘 수 있다.

### 2.3 엔터티 분류

| 분류 | 설명                                  | 예시                   |
| ---- | ------------------------------------- | ---------------------- |
| 기준 | 업무 기준이 되는 마스터 데이터        | 사용자, 조직, 공통코드 |
| 업무 | 업무 수행 과정에서 발생하는 데이터    | 계약, 프로젝트, 청구   |
| 관계 | 다대다 관계를 해소하기 위한 데이터    | 사용자역할, 직원기술   |
| 이력 | 변경 또는 처리 이력을 관리하는 데이터 | 계약변경이력, 접속로그 |
| 집계 | 집계 또는 마감 결과를 저장하는 데이터 | 월별원가, 운영실적     |

## 5. 엔터티 검토 대상

### 5.1 통합 검토 대상

| 대상                     | 검토 의견                                                                                                                                       |
| ------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| 사용자 / 직원            | 사용자는 로그인 계정, 직원은 인력관리 대상이다. 내부 직원만 로그인한다면 통합 가능하나, 외부 사용자 가능성을 고려하면 분리하는 것이 안정적이다. |
| 계약 / 유지보수계약      | 유지보수계약은 계약의 한 유형으로 볼 수 있다. SM 특화 속성이 많지 않다면 계약 엔터티로 통합 가능하다.                                           |
| 원가 / 월별원가          | 월별 원가 집계가 핵심이면 원가 엔터티를 월별 원가 구조로 설계할 수 있다.                                                                        |
| 손익 / 매출현황 / 미수금 | 대부분 조회성 또는 집계성 데이터이므로 물리 테이블 여부를 검토한다.                                                                             |
| 기술등급 / 공통코드      | 기술등급이 단순 코드라면 공통코드로 관리 가능하다.                                                                                              |

### 5.2 추가 검토 대상

| 대상           | 검토 의견                                                                         |
| -------------- | --------------------------------------------------------------------------------- |
| 승인 이력      | 예산 변경, 계약 변경 등 승인 절차가 구체화되면 공통 승인 엔터티가 필요할 수 있다. |
| 업무 상태 이력 | 계약상태, 프로젝트상태, 청구상태 등의 변경 이력을 별도 관리할지 검토한다.         |
| 파일 첨부 구조 | 계약문서, 산출물, 제안서 등은 첨부파일 공통 구조로 통합 관리할 수 있다.           |
| 업무 공통 메모 | 고객, 영업, 계약, 프로젝트 단위 메모를 범용 메모 구조로 관리할 수 있다.           |
